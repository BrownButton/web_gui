# 03. 웹 기술 설명

> 이 문서는 C#/펌웨어 개발자를 위한 웹 기술 입문 설명입니다.

---

## 1. 웹 앱 vs 데스크톱 앱 비교

| 항목 | C# WinForm/WPF | 이 프로젝트 (웹앱) |
|------|---------------|-------------------|
| 실행 환경 | Windows OS (.NET 런타임) | Chrome/Edge 브라우저 |
| 설치 | 설치 파일 필요 | 없음 (파일 열기만) |
| UI 정의 | XAML / Designer | HTML + CSS |
| 로직 | C# | JavaScript |
| 빌드 | .NET 컴파일러 | 없음 (소스 그대로 실행) |
| 배포 | Setup.exe 배포 | 파일 복사/교체 |
| 시리얼 통신 | System.IO.Ports.SerialPort | Web Serial API |
| 디버깅 | Visual Studio 디버거 | 브라우저 개발자 도구 (F12) |

## 2. 웹 3대 요소 (HTML, CSS, JavaScript)

웹 앱은 세 가지 파일 형식으로 구성됩니다.
C# 앱으로 비유하면 다음과 같습니다:

### HTML — 구조 (뼈대)

C# WPF에서 `Window.xaml`이 UI 요소를 정의하듯,
HTML은 페이지의 구조를 정의합니다.

```html
<!-- index.html 발췌 -->
<div class="device-card">
  <span class="device-name">EC-FAN #1</span>
  <span class="actual-speed">1200 RPM</span>
  <button class="setpoint-btn">설정</button>
</div>
```

### CSS — 디자인 (외형)

C# WPF에서 Style/Template이 외형을 정의하듯,
CSS는 색상, 크기, 레이아웃을 정의합니다.

```css
/* styles.css 발췌 */
.device-card {
  background-color: #1e1e2e;
  border-radius: 8px;
  padding: 16px;
  display: flex;
  flex-direction: column;
}
```

### JavaScript — 동작 (로직)

C#에서 코드비하인드(`.cs` 파일)가 이벤트를 처리하듯,
JavaScript는 버튼 클릭, 데이터 처리, 통신 등 모든 동작을 담당합니다.

```javascript
// app.js 발췌
button.addEventListener('click', () => {
  writeRegister(device.slaveId, 0xD001, newSetpoint);
});
```

## 3. 이 프로젝트의 파일 구조

```
web_gui/
├── index.html          ← HTML (구조)
├── styles.css          ← CSS (디자인)
├── app.js              ← JavaScript (메인 로직, ~11,000 라인)
├── modbus.js           ← JavaScript (Modbus 프로토콜 라이브러리)
├── simulator.js        ← JavaScript (가상 슬레이브 장치)
├── os-test-manager.js  ← JavaScript (자동화 테스트 프레임워크)
└── parameters.csv      ← CSV (레지스터 파라미터 정의 데이터)
```

### 외부 프레임워크 없음 (Pure Vanilla JS)

React, Vue, Angular 같은 유명 웹 프레임워크를 사용하지 않고
순수한 HTML/CSS/JavaScript만 사용했습니다.

이유:
- 추가 빌드 도구 없이 파일만으로 실행 가능
- 의존성이 없어 유지보수 단순
- AI와의 협업에서 프레임워크 특성을 고려할 필요 없음

## 4. Single Page Application (SPA) 개념

이 앱은 **SPA(Single Page Application)** 방식입니다.

### 전통적인 웹사이트

페이지를 이동할 때마다 서버에서 새 HTML을 받아옵니다.
마치 새 화면을 로드하는 것처럼 깜빡임이 발생합니다.

### SPA 방식 (이 프로젝트)

처음 한 번만 HTML을 로드하고, 이후 탭 전환은 JavaScript로 요소를 숨기고 보여줍니다.
깜빡임 없이 빠르게 탭이 전환됩니다.

```javascript
// 탭 전환 예시 (app.js 내부 동작)
function showPage(pageName) {
  // 모든 페이지 숨기기
  document.querySelectorAll('.page').forEach(p => p.style.display = 'none');
  // 선택한 페이지만 보이기
  document.getElementById(pageName).style.display = 'block';
}
```

C# WPF의 `Frame.Navigate()` 또는 탭 컨트롤과 유사한 개념입니다.

## 5. 데이터 저장 (localStorage)

C# 앱에서는 설정을 레지스트리 또는 config 파일에 저장합니다.
웹 앱에서는 **localStorage**라는 브라우저 내장 저장소를 사용합니다.

```javascript
// 장치 목록 저장 (앱 종료 후에도 유지됨)
localStorage.setItem('devices', JSON.stringify(this.devices));

// 다음 실행 시 복원
const saved = localStorage.getItem('devices');
this.devices = saved ? JSON.parse(saved) : [];
```

| 특성 | C# (Registry/INI) | 웹 (localStorage) |
|------|------------------|--------------------|
| 저장 위치 | Windows 레지스트리 or 파일 | 브라우저별 로컬 저장소 |
| 용량 | 무제한 (파일 기준) | 약 5~10MB |
| 접근 방식 | Registry API / File I/O | `localStorage.getItem/setItem` |
| 초기화 | 파일 삭제 | 브라우저 캐시 삭제 |

## 6. 브라우저 실행 방식

C# 앱은 .exe를 실행하면 바로 실행됩니다.
웹 앱은 브라우저가 HTML 파일을 해석하여 실행합니다.

### 실행 방법

```
index.html 파일을 Chrome 또는 Edge로 열기
(더블클릭 또는 드래그 앤 드롭)
```

또는 VS Code의 Live Server 확장 사용:
```
VS Code에서 index.html 우클릭 → "Open with Live Server"
→ http://localhost:5500/index.html 자동 열림
```

### 주의: Web Serial은 보안 컨텍스트 필요

Web Serial API는 **보안 컨텍스트**(Secure Context)에서만 동작합니다.
- `https://` URL ✓
- `http://localhost/` ✓ (로컬 테스트용)
- `file:///` (파일 직접 열기) ✗ (일부 브라우저에서 제한)

따라서 파일을 더블클릭해서 열면 안 되고,
**로컬 웹 서버를 통해 접근**해야 합니다.

```
방법 1: VS Code의 "Live Server" 확장 사용
방법 2: Python 간단 서버: python -m http.server 8080
         → http://localhost:8080/index.html
방법 3: 실제 웹 서버(Nginx, Apache)에 배포
```

## 7. JavaScript 비동기 처리 (async/await)

C# 4.5+의 `async/await`와 거의 동일한 개념이 JavaScript에도 있습니다.
시리얼 통신처럼 시간이 걸리는 작업은 비동기로 처리합니다.

### C# 예시

```csharp
// C# async/await
async Task<string> ReadRegisterAsync(int address) {
    await serialPort.WriteAsync(frame);
    var response = await serialPort.ReadAsync(timeout);
    return ParseResponse(response);
}
```

### JavaScript 예시 (이 프로젝트)

```javascript
// JavaScript async/await (거의 동일한 문법)
async function readRegisterWithTimeout(slaveId, address) {
    const frame = ModbusRTU.buildReadHoldingFrame(slaveId, address, 1);
    await this.writer.write(frame);
    const response = await waitForResponse(slaveId, timeout);
    return parseResponse(response);
}
```

비동기 처리가 중요한 이유:
- 응답을 기다리는 동안 UI가 멈추지 않음
- 여러 장치를 순차적으로 폴링 가능
- 타임아웃 처리 용이

## 8. 이벤트 드리븐 프로그래밍

C# WinForm/WPF처럼 JavaScript도 이벤트 기반으로 동작합니다.

| C# | JavaScript |
|----|------------|
| `button.Click += Handler` | `button.addEventListener('click', handler)` |
| `textBox.TextChanged += Handler` | `input.addEventListener('input', handler)` |
| `timer.Tick += Handler` | `setInterval(handler, intervalMs)` |
| `serialPort.DataReceived += Handler` | `reader.read()` 루프 (Web Serial) |

```javascript
// 버튼 클릭 이벤트 예시
document.getElementById('connectBtn').addEventListener('click', async () => {
    await this.connectSerial();  // 연결 버튼 클릭 시 시리얼 연결
});

// 타이머 (폴링 루프)
this.autoPollingTimer = setInterval(() => {
    this.pollNextDeviceSequential();  // 100ms마다 실행
}, 100);
```

## 9. 개발자 도구 (F12)

Chrome/Edge에서 F12를 누르면 개발자 도구가 열립니다.
Visual Studio의 디버거와 유사한 기능을 제공합니다.

| Visual Studio | 브라우저 F12 |
|---------------|------------|
| 중단점(Breakpoint) | Sources 탭 → 클릭으로 중단점 설정 |
| 변수 감시 | Console 탭 → `window.dashboard.devices` 입력 |
| 출력창 | Console 탭 → `console.log()` 출력 |
| 네트워크 | Network 탭 |
| 성능 분석 | Performance 탭 |

```javascript
// 개발자 도구 콘솔에서 바로 실행 가능
window.dashboard.devices           // 장치 목록 확인
window.dashboard.commandQueue      // 큐 상태 확인
window.dashboard.isPolling         // 폴링 상태 확인
```

# 1-2. Broadcast 동작 시험

| 항목 | 내용 |
|------|------|
| **분류** | RS485 |
| **No** | 1-2 |

## 시험 목적

Node ID 0 (Broadcast)으로 명령 전송 시 RS485 버스에 연결된 모든 드라이브에 동시 적용되는지 확인한다.

## 적용 모델

- EC-FAN ✓

## 시험 장비

- EC FAN 2EA 이상 (각각 다른 Node ID 설정)
- USB to RS485 Converter
- Hercules v3.2.8 (테스트 프로그램)

## 시험 방법

1. EC FAN 2대를 RS485 버스에 병렬 연결한다.
2. 각 드라이브에 서로 다른 Node ID를 설정한다. (예: Node 1, Node 2)
   - Node ID 설정 방법은 RS485 No. 1-3 시험 참조
3. USB to RS485 Converter를 이용하여 EC FAN과 노트북을 연결한다.
4. Hercules v3.2.8 테스트 프로그램을 실행한다.
5. Serial 탭에서 통신 설정을 진행한다.
   - Baud: 19200bps, Data Size: 8, Parity: even
6. Open 버튼을 클릭하여 COM Port 연결을 확인한다.
7. Broadcast ID(0x00)로 Set Point 명령을 전송한다.

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|-----|------|------|------|------|------|------|------|
| 내용 | 0x00 | 0x06 | 0xD0 | 0x01 | 0x19 | 0x9A | CRC Hi | CRC Lo |

- Broadcast ID: 0x00 (응답 없음, 단방향 전송)
- FC 0x06: Write Single Register
- 주소 0xD001: Set Point

8. 연결된 모든 드라이브가 명령에 반응하는지 (모터 구동 등) 확인한다.
9. 각 드라이브의 Set Point 파라미터 값을 개별적으로 읽어 동일하게 설정되었는지 확인한다.

## 판정기준

- Broadcast ID(0x00)로 전송한 명령이 버스에 연결된 모든 드라이브에 동시 적용

## 시험결과

- [ ] 합격 / 불합격

## 기타

- Broadcast 명령은 드라이브로부터 응답 없음 (단방향 전송)
- 시험 전 각 드라이브에 고유 Node ID가 설정되어 있어야 함

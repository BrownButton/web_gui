# 1-1. 기본 연결 동작 시험 및 Baudrate / Parity / Stop Bit 변경 [개선]

| 항목 | 내용 |
|------|------|
| **분류** | RS485 |
| **No** | 1-1 |
| **버전** | 개선 v1 |

## 시험 목적

EC FAN과 RS485 Modbus 통신 연결이 정상적으로 이루어지는지 확인하고,
드라이브가 지원하는 **Baudrate × Parity × Stop Bit의 모든 조합**에서 통신이 정상 동작하는지 검증한다.
시험 완료 후 초기 통신 설정으로 완전히 복원한다.

## 적용 모델

- EC-FAN ✓

## 시험 장비

- EC FAN 1EA
- USB to RS485 Converter
- EC FAN Configurator (EC FAN Control 프로그램)

> ⚠️ **단일 장비 전제**: 이 시험은 장비 1대만 연결된 환경에서 진행한다.
> 통신 설정(Baudrate / Parity / Stop Bit)을 변경하면 동일 RS485 버스에 연결된
> 다른 장비는 설정 불일치로 응답하지 않게 된다.
> 멀티 장비 환경에서는 반드시 해당 장비만 버스에 연결하고 나머지는 분리 후 진행한다.

---

## 시험 전 조건 (Preconditions)

| 항목 | 초기값 | 비고 |
|------|--------|------|
| Baudrate | **19200 bps** | 공장 출하 기본값 |
| Parity | **Even** | 공장 출하 기본값 |
| Stop Bit | **1** | Parity Even/Odd 시 1 |
| Node ID | 1 | |
| Motor Status | 정지, 알람 없음 | |

> 이전 시험에서 설정이 변경된 경우 반드시 초기화 후 진행.

---

## 통신 확인 방법 및 파라미터 선택

각 케이스에서 통신 정상 여부는 아래 파라미터 중 **하나 이상**을 Modbus 명령으로 읽어
정상 응답이 수신되면 합격으로 판정한다.

**기본 확인 파라미터:**

| 파라미터 | FC | 레지스터 주소 | 비고 |
|----------|----|-------------|------|
| Device Name | FC03 | (기기별 확인) | |
| Main SW Ver | FC03 | (기기별 확인) | |
| Main Boot Ver | FC03 | (기기별 확인) | |
| Inverter SW Ver | FC03 | (기기별 확인) | |
| Inverter Boot Ver | FC03 | (기기별 확인) | |

> 시험자가 위 중 일부를 선택하거나 다른 레지스터를 지정하여 진행할 수 있다.
> 이 시험의 목적은 **통신 가능 여부** 확인이며, 파라미터의 값 자체는 판정 기준이 아니다.

---

## 지원 통신 설정 매트릭스

아래는 드라이브가 지원하는 통신 설정 목록이다.
**사양서 확인 후 미지원 항목은 결과표에 "미지원"으로 기록하고 해당 케이스를 생략한다.**

| 항목 | 지원 값 | 기본값 |
|------|---------|--------|
| Baudrate | 9600 / **19200** / 38400 / 57600 / 115200 | 19200 |
| Parity | **Even** / Odd / None | Even |
| Stop Bit | **1** / 2 | 1 |

### 통신 조합 규칙

| Parity | 권장 Stop Bit | 비고 |
|--------|--------------|------|
| Even | 1 | Modbus RTU 표준 |
| Odd | 1 | Modbus RTU 표준 |
| None | 2 | Modbus RTU 표준 (Stop Bit=1도 드라이브 지원 시 추가 시험) |

## 시험 절차

### 공통 변경 절차 (각 케이스에 적용)

```
① Setting 메뉴 → Baudrate / Parity / Stop Bit를 해당 케이스 값으로 설정
② Save to Memory
③ 전원 Off → 3초 대기 → 전원 On (물리적 재투입)
④ 프로그램에서 해당 케이스 설정으로 재접속
⑤ 통신 확인 파라미터 읽기 → 응답 정상 수신 여부 기록
⑥ 초기값(19200 / Even / Stop Bit 1)으로 복구 → Save → 전원 재투입 → 정상 접속 확인 후 다음 케이스 진행
```

---

### [케이스 0] 초기값 기본 접속 확인 (Baseline)

- 설정: Baudrate 19200 / Parity Even / Stop Bit 1
- 프로그램 접속 후 통신 확인 파라미터 읽기
- 이 케이스의 값을 이후 비교 기준으로 사용

---

### [케이스 1 ~ N] 전체 조합 시험

아래 표의 모든 케이스를 순서대로 시험한다.
각 케이스 후 반드시 초기값으로 복구 후 다음 케이스 진행.

| 케이스 | Baudrate | Parity | Stop Bit | 비고 |
|--------|----------|--------|----------|------|
| B-1 | 9600 | Even | 1 | |
| B-2 | 38400 | Even | 1 | |
| B-3 | 57600 | Even | 1 | |
| B-4 | 115200 | Even | 1 | |
| P-1 | 19200 | Odd | 1 | |
| P-2 | 19200 | None | 2 | Modbus RTU 표준 |
| P-3 | 19200 | None | 1 | 드라이브 지원 시만 진행 |
| C-1 | 9600 | Odd | 1 | |
| C-2 | 9600 | None | 2 | |
| C-3 | 38400 | Odd | 1 | |
| C-4 | 38400 | None | 2 | |
| C-5 | 57600 | Odd | 1 | |
| C-6 | 57600 | None | 2 | |
| C-7 | 115200 | Odd | 1 | |
| C-8 | 115200 | None | 2 | |

> B-: Baudrate만 변경 (Parity=Even 고정)
> P-: Parity만 변경 (Baudrate=19200 고정)
> C-: 조합 케이스

---

### [최종] 초기값 복원 확인

모든 케이스 완료 후:

1. 현재 통신 설정이 초기값(19200 / Even / Stop Bit 1)인지 드라이브 Setting 메뉴에서 확인한다.
2. 초기값과 다를 경우 설정 후 Save to Memory → 전원 재투입한다.
3. 프로그램을 초기값으로 접속하여 통신 확인 파라미터 정상 수신을 확인한다.

---

## 판정기준

| 케이스 | 판정 조건 |
|--------|----------|
| 케이스 0 | 초기값으로 정상 접속 및 파라미터 응답 수신 |
| 케이스 B-1 ~ B-4 | 각 Baudrate 변경 후 재접속 및 정상 응답 수신 |
| 케이스 P-1 ~ P-3 | 각 Parity/Stop Bit 변경 후 재접속 및 정상 응답 수신 |
| 케이스 C-1 ~ C-8 | 각 조합 변경 후 재접속 및 정상 응답 수신 |
| 최종 복원 | 초기값(19200 / Even / 1)으로 정상 복원 및 통신 확인 |

**합격 조건**: 지원하는 모든 케이스에서 정상 응답 수신 + 최종 복원 완료

## 기타

- 각 변경 케이스 후 초기값 복구 및 정상 접속 확인까지 완료해야 해당 케이스 종료

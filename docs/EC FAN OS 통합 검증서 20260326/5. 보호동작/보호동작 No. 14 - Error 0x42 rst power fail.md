# 보호동작 No. 14 - Error 0x42 rst power fail

## 시험 항목

주 전원(RST) 결상 시 드라이브가 보호 동작(알람)을 수행하고 Motor Status [0xD011]의 PHA 비트(Bit 0) 및 FB 비트(Bit 4)가 Set되는지 확인하는 시험이다.

## 시험 목표

R, S, T 각 상의 결상 조건을 만들어 드라이브가 보호 동작을 수행하고 Motor Status [0xD011]의 PHA 비트(Bit 0) 및 FB 비트(Bit 4)가 Set되는지 확인한다.

## 시험 기준

### 사전 조건

- RS485 인터페이스를 통해 드라이브와 호스트 PC 연결 완료
- 각 상별 결상 가능한 전원 배선 접근 가능
- Slave ID: 0x01 기준

### 시험 절차

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. 아래 표의 각 케이스에 따라 주 전원을 결상시킨다.

| Test Case | R | S | T |
|-----------|---|---|---|
| Case 1 | O | O | X |
| Case 2 | O | X | O |
| Case 3 | X | O | O |
| Case 4 | O | X | X |
| Case 5 | X | X | O |
| Case 6 | X | O | X |
| Case 7 | X | X | X |

4. 전원을 켠 후 Motor Status [0xD011]을 읽어 에러 비트를 확인한다.

**Motor Status 읽기 명령 — FC(04) Read Input Register**

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x04 | 0xD0 | 0x11 | 0x00 | 0x01 | 0x59 | 0x0F |

→ `01 04 D0 11 00 01 59 0F`

**Motor Status 비트맵 (확인 비트)**

| 비트 | 이름 | 설명 |
|------|------|------|
| 0 | PHA | 주 전원 결상 |
| 4 | FB | 팬 에러 (에러 상태 활성화) |

- 예상 Motor Status 값: 0x0011

### 판정 기준

각 케이스에서 결상 조건 인가 후 Motor Status [0xD011] = **0x0011** (PHA Bit 0 + FB Bit 4 Set) 확인

| 케이스 | 결상 상 | 판정 조건 |
|--------|---------|----------|
| Case 1 | T 상 결상 | Motor Status = 0x0011 |
| Case 2 | S 상 결상 | Motor Status = 0x0011 |
| Case 3 | R 상 결상 | Motor Status = 0x0011 |
| Case 4 | S, T 결상 | Motor Status = 0x0011 |
| Case 5 | R, T 결상 | Motor Status = 0x0011 |
| Case 6 | R, S 결상 | Motor Status = 0x0011 |
| Case 7 | R, S, T 전상 결상 | Motor Status = 0x0011 |

**합격 조건**: 7가지 결상 케이스 모두에서 보호 동작 확인

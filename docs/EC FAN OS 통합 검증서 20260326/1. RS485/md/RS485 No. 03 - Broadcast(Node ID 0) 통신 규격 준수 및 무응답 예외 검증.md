# RS485 No. 03 - Broadcast (Node ID 0) 통신 규격 준수 및 무응답(No-Response) 예외 검증

## 시험 항목

Broadcast(Node ID 0) 통신 규격 준수 및 무응답 예외 검증

## 시험 목표

- Broadcast ID(0x00)로 전송한 쓰기 명령이 버스에 연결된 모든 드라이브에 동시 적용됨을 확인한다.
- Broadcast 명령에 대해 드라이브가 응답을 반환하지 않음(No-Response)을 확인한다.
- 응답 없음 상태에서 호스트가 타임아웃 처리를 올바르게 수행하는지 확인한다.

## 시험 기준

### 사전 조건

| 항목 | 값 | 비고 |
|------|-----|------|
| 드라이브 수 | 2대 이상 | RS485 버스에 병렬 연결 |
| Node ID | 각기 다른 ID (예: 1, 2) | |
| Baudrate | 19200 / Even / Stop Bit 1 | |
| Motor Status | 정지, 알람 없음 | |

### 시험 절차

#### [케이스 1] Broadcast 명령 동시 적용 확인

| 단계 | 동작 | Modbus 프레임 |
|------|------|--------------|
| 1 | 각 드라이브 개별 접속 후 현재 Set Point 읽기 | `01 03 D0 01 00 01 CRC` / `02 03 D0 01 00 01 CRC` |
| 2 | Broadcast(0x00)로 Set Point 값 쓰기 | `00 06 D0 01 19 9A CRC` |
| 3 | 드라이브 Node 1로 Set Point 읽기 | `01 03 D0 01 00 01 CRC` |
| 4 | 드라이브 Node 2로 Set Point 읽기 | `02 03 D0 01 00 01 CRC` |

- Broadcast ID: `0x00`
- FC `0x06`: Write Single Register
- 레지스터 `0xD001`: Set Point
- 값 `0x199A`: 약 25% (Open-loop 기준 0x199A ≈ 6554 → ~10%)

> 단계 2 전송 후 응답 수신 대기 없이 일정 시간(드라이브 처리 시간, 약 100ms) 대기 후 단계 3, 4 진행.

#### [케이스 2] Broadcast 무응답(No-Response) 확인

| 단계 | 동작 |
|------|------|
| 1 | Broadcast(0x00)로 FC06 명령 전송 |
| 2 | 응답 수신 대기 (타임아웃: 500ms) |
| 3 | 호스트 측 타임아웃 발생 여부 기록 |

- Modbus RTU 규격: Broadcast 명령에 대해 슬레이브는 응답하지 않음
- 호스트(예지보전 툴)가 타임아웃 처리 후 오류 없이 다음 동작을 수행하는지 확인

#### [케이스 3] Broadcast 읽기 명령 동작 확인 (옵션)

| 단계 | 동작 | Modbus 프레임 |
|------|------|--------------|
| 1 | Broadcast(0x00)로 FC03 읽기 명령 전송 | `00 03 D0 01 00 01 CRC` |
| 2 | 응답 수신 대기 |

- Modbus RTU 규격: Broadcast 읽기는 정의되지 않음 (드라이브가 응답하지 않아야 함)
- 응답 없음 또는 드라이브별 동작을 기록

### 판정 기준

| 케이스 | 판정 조건 |
|--------|----------|
| 케이스 1 | Broadcast 쓰기 명령 후 모든 드라이브에 동일한 값이 설정됨 |
| 케이스 2 | Broadcast 명령 후 드라이브가 응답을 반환하지 않음 (호스트 타임아웃 발생) |
| 케이스 3 (옵션) | Broadcast 읽기 명령에 드라이브가 응답하지 않음 |

**합격 조건**: Broadcast 명령이 모든 드라이브에 동시 적용 + No-Response 규격 준수 확인

# RS485 No. 02 - Node ID 설정 및 예외 검증

## 시험 항목

RS485 Modbus Node ID(Fan Address)를 변경하고, 변경된 ID로 통신이 정상 동작하는지 확인한다. 또한 유효 범위를 벗어난 ID 입력 시 예외 처리가 올바르게 동작하는지 검증한다.

## 시험 목표

- Fan Address(Node ID) 레지스터 [0xD100] 쓰기 → Save to Memory → 전원 재투입 후 새 ID로 통신이 성립됨을 확인한다.
- 유효 범위(1 ~ 247) 이외의 값(0, 248~255) 입력 시 드라이브가 설정을 거부하거나 기존값을 유지함을 확인한다.

## 시험 기준

### 사전 조건

| 항목 | 초기값 | 비고 |
|------|--------|------|
| Node ID | 1 | |
| Baudrate | 19200 | Even / Stop Bit 1 |
| Motor Status | 정지, 알람 없음 | |

### 시험 절차

#### [케이스 1] 정상 Node ID 변경

| 단계 | 동작 | Modbus 프레임 (Node 1 기준) |
|------|------|---------------------------|
| 1 | 현재 Node ID 읽기 (확인) | `01 03 D1 00 00 01 CRC` |
| 2 | Fan Address를 7로 변경 | `01 06 D1 00 00 07 CRC` |
| 3 | Save to Memory | `01 06 D0 00 00 01 CRC` |
| 4 | 전원 Off → 3초 → 전원 On | — |
| 5 | Node 7로 재접속 후 Fan Address 읽기 | `07 03 D1 00 00 01 CRC` |
| 6 | Node ID를 1로 복원 | `07 06 D1 00 00 01 CRC` |
| 7 | Save to Memory → 전원 재투입 → Node 1로 정상 접속 확인 | `01 06 D0 00 00 01 CRC` |

> Save to Memory: 레지스터 0xD000에 0x0001 쓰기 (FC06)

#### [케이스 2] 유효 범위 경계값 검증

| 케이스 | 입력 Node ID | 예상 동작 |
|--------|------------|----------|
| 경계 하한 | 1 | 정상 설정 |
| 경계 상한 | 247 | 정상 설정 |
| 범위 초과 (하) | 0 | 설정 거부 또는 기존값 유지 |
| 범위 초과 (상) | 248 | 설정 거부 또는 기존값 유지 |

각 케이스에서 설정 후 재투입 없이 또는 재투입 후 해당 ID로 통신을 시도하여 결과를 기록한다.

### 판정 기준

| 케이스 | 판정 조건 |
|--------|----------|
| 케이스 1 | Fan Address 변경 → Save → 재투입 후 새 ID로 정상 통신 성립 |
| 케이스 2 (경계) | 1, 247로 정상 설정 및 통신 성립 |
| 케이스 2 (초과) | 0, 248 입력 시 설정 거부 또는 기존값 유지 확인 |

**합격 조건**: 정상 범위 내 ID 변경 후 통신 성립 + 범위 초과 시 예외 처리 확인

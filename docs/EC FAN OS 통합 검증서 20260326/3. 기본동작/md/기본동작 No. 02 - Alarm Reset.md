# 기본동작 No. 02 - Alarm Reset

## 시험 항목

Alarm Reset

## 시험 목표

- 드라이브가 알람(Fault) 상태에 진입한 후, Alarm Reset 명령(레지스터 0x800E에 0x0001 write)을 전송하면 Motor Status [0xD011]의 에러 비트가 Clear됨을 확인한다.
- 알람 해제 후 정상 운전 가능 상태로 복귀함을 확인한다.

## 시험 기준

### 사전 조건

- RS485 인터페이스를 통해 드라이브와 호스트 PC 연결 완료
- 알람 발생 상태 (보호동작 시험 중 발생한 알람 활용 가능)
- Slave ID: 0x01 기준

### 시험 절차

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. 알람 상태를 유발한다.
   - 예시) 모터 구속 상태에서 구동 명령 전송, 또는 보호동작 시험 중 발생한 알람 활용
4. Motor Status [0xD011]을 읽어 에러 비트(Bit 4: FB)가 Set 되어 있는지 확인한다.

**Motor Status 읽기 명령 — FC(04) Read Input Register**

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x04 | 0xD0 | 0x11 | 0x00 | 0x01 | 0x59 | 0x0F |

→ `01 04 D0 11 00 01 59 0F`

5. Alarm Reset 명령을 전송한다.

**Alarm Reset 명령 — FC(06) Write Single Register**
- 레지스터: Alarm Reset [0x800E]
- 값: 0x0001

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | 0x80 | 0x0E | 0x00 | 0x01 | CRC Lo | CRC Hi |

→ `01 06 80 0E 00 01 19 C8`

> ⚠️ Alarm Reset은 0xD000이 아닌 **0x800E** 레지스터에 전송한다.

6. Alarm Reset 후 Motor Status [0xD011]을 다시 읽어 에러 비트가 Clear 되었는지 확인한다.
7. 정상 운전 명령을 전송하여 구동 가능 여부를 확인한다.

### 판정 기준

- Alarm Reset 명령 전송 후 Motor Status의 에러 비트(FB, 기타 에러 비트)가 Clear됨
- 알람 해제 후 정상 운전 가능 상태로 복귀

**합격 조건**: 위 판정 조건 모두 만족

시험 항목 : [RS485 Modbus 기반 Alarm Reset (FC: 0x06) 조건부 해제 무결성 및 Anti-Auto-Restart(안전 복구) 통합 검증]

시험 목적 : Modbus FC 0x06 명령으로 Alarm Reset 레지스터 [0x800E]에 해제 코드(0x0001) 전송 시 정상적으로 에러가 Clear 되는지 확인한다. 또한, 물리적 에러 원인이 잔존할 경우의 리셋 거부(Latching), 비정상 리셋 코드 입력 시의 예외 처리, 그리고 알람 해제 직후 사용자의 명시적인 재가동 명령 없이 스스로 모터가 급발진(Auto-Restart)하지 않는지 종합적인 하드웨어/소프트웨어 안전성을 검증한다.

시험 방법 :

<Phase 1. 초기 상태 연결 및 정상 구동 중 Alarm Reset 인가 검증 (Base Setup & No-op Test)>
RS485 인터페이스를 통해 드라이브(Node ID: 0x01)와 PC를 연결하고, 통신 프로그램(19200bps, Even, 8, 1)을 오픈한다.
드라이브에 Run 명령을 내려 모터를 정상 속도(예: 1000 RPM)로 회전시킨다.
알람(Fault)이 없는 정상 구동 상태에서 기습적으로 Alarm Reset 명령을 전송(Write Single Register, FC: 0x06)한다.
송신(TX) 프레임: 01 06 80 0E 00 01 19 C8
모터가 출렁이거나(PWM Drop) 통신이 끊기지 않고 부드럽게 정상 회전을 유지하는지 물리적으로 관찰한다.

<Phase 2. 물리적 원인 잔존 상태에서의 Alarm Reset 예외 검증 (Unresolved Fault Test)>
모터 회전 중, 강제로 드라이브 입력 전압을 한계치 이상(또는 인위적 모터 구속/과전류 상태)으로 올려 하드웨어 알람(예: OVP 또는 Over-Current)을 유발시킨다.
모터가 급정지(Trip)된 후, Motor Status [0xD011]를 읽어(FC: 0x04) 에러 비트(Bit 4: FB 또는 해당 Fault Bit)가 Set 상태인지 확인한다.
송신(TX) 프레임: 01 04 D0 11 00 01 59 0F
[핵심 조작] 입력 전압을 낮추지 않고(즉, 에러 원인이 그대로 남아있는 과전압 상태 유지), 통신으로 Phase 1의 Alarm Reset 명령(01 06 80 0E 00 01 19 C8)을 강제 송신한다.
다시 Motor Status [0xD011]를 읽어 에러 비트가 여전히 살아있는지(Reset 거부) 확인한다.

<Phase 3. 원인 제거 후 비정상 리셋 코드 입력 예외 검증 (Negative Test)>
드라이브 입력 전압을 정상(정격 전압)으로 낮추거나, 구속 상태를 해제하여 물리적인 에러 원인을 완전히 제거한다.
아직 알람(Trip) 상태가 유지 중인 드라이브에, 규정된 리셋 코드(0x0001)가 아닌 쓰레기 값(예: 0x0000, 0x0002, 0xFFFF)을 강제 송신 시도한다.
예외 송신(TX) 프레임 (0xFFFF 주입 시): 01 06 80 0E FF FF 28 0A
Motor Status [0xD011]를 읽어 펌웨어가 비정상 리셋 코드를 거부하고 알람 상태를 굳건히 유지하는지 확인한다.

<Phase 4. 정상 Alarm Reset 수행 및 Anti-Auto-Restart (안전 대기) 검증 (Recovery & Safety Test)>
정상적인 Alarm Reset 코드(0x0001)를 레지스터 [0x800E]에 송신(01 06 80 0E 00 01 19 C8)한다.
Motor Status [0xD011]를 읽어 에러 비트가 완벽히 Clear(0x0000) 되었는지 확인한다.
[안전성 핵심 관찰] 알람 발생 전 인가되어 있던 Run 명령이 아직 상위 제어기(또는 레지스터)에 남아있더라도, 알람이 해제되자마자 모터가 사용자의 추가 조작 없이 스스로 회전을 시작(Auto-Restart)하는지 5초간 지켜본다.
모터가 정지 상태를 유지한다면, 새롭게 Stop 명령을 내렸다가 다시 Run 명령을 전송하여 모터가 정상적으로 재구동되는지 최종 확인한다.

판정기준 :
[Phase 1 판정 - 정상 구동 중 리셋]
에러가 없는 상태에서 Alarm Reset 명령 수신 시, 펌웨어는 정상적으로 Echo(ACK) 응답을 주되 내부 제어 루프나 통신에 어떠한 영향(Hiccup)도 주지 않아야 한다.

[Phase 2 판정 - 물리적 원인 잔존 시 (하드웨어 보호 핵심)]
에러의 근본 원인(과전압, 과전류 등 물리적 위험 상태)이 제거되지 않았다면, Alarm Reset 명령을 아무리 전송해도 에러 비트(Motor Status [0xD011])는 절대 Clear 되어서는 안 된다. (안전 확보 시까지 Fault 상태 Latching 유지 필수)

[Phase 3 판정 - 예외 처리]
0x0001 이외의 값 입력 시 파라미터 갱신을 거부하고 Modbus Exception Code 0x03 (Illegal Data Value) 프레임(예: 01 86 03 02 61)을 반환해야 하며, 알람 상태는 그대로 유지되어야 합격 처리한다.

[Phase 4 판정 - 알람 해제 및 안전 복구 (Safety Interlock)]
에러 원인 제거 후 정상 리셋 코드를 전송하면, Motor Status가 정상 상태로 Clear 되어야 한다.
[안전 규격 합격 필수 조건] 알람이 해제되었다고 해서 과거의 Run 지령을 물고 즉각 급가속(Auto-Restart)하면 작업자 상해 위험이 있으므로 즉시 불합격(Fail) 처리한다.
시스템은 반드시 정지 대기(Ready) 상태로 진입해야 하며, 상위 마스터가 새로운 Run/Stop Edge(명령 토글)를 주었을 때만 재가동되어야 최종 합격이다.

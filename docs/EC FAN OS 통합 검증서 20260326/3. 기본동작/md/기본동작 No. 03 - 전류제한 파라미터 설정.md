# 기본동작 No. 03 - 전류제한 파라미터 설정

## 시험 항목

전류제한 파라미터 설정

## 시험 목표

- Maximum Coil Current [0xD13B] 파라미터에 전류 제한값을 쓰고 정상 기록됨을 확인한다.
- EEPROM Save 후 전원 재투입 시에도 설정값이 유지됨을 확인한다.
- 구동 중 모터 전류가 설정 제한값을 초과하지 않는지 확인한다.

## 시험 기준

### 사전 조건

- RS485 인터페이스를 통해 드라이브와 호스트 PC 연결 완료
- Slave ID: 0x01 기준

### 시험 절차

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. 현재 Max Coil Current [0xD13B] 값을 FC(03)으로 읽어 확인한다.

**Max Coil Current 읽기 명령 — FC(03) Read Holding Register**

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x03 | 0xD1 | 0x3B | 0x00 | 0x01 | CRC Lo | CRC Hi |

→ `01 03 D1 3B 00 01 14 30`

4. 전류 제한값을 변경하여 FC(06)으로 기록한다.

**Max Coil Current 쓰기 명령 — FC(06) Write Single Register**
- 레지스터: Maximum Coil Current [0xD13B]
- 값: 설정할 전류 제한값 (단위: 제품 사양 참고)

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | 0xD1 | 0x3B | Value Hi | Value Lo | CRC Lo | CRC Hi |

5. EEPROM Save를 실행하여 변경된 값을 저장한다.

**EEPROM Save 명령 — FC(06) Write Single Register**
- 레지스터: Reset [0xD000]
- 값: 0x0004 (EEPROM to RAM Save)

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | 0xD0 | 0x00 | 0x00 | 0x04 | CRC Lo | CRC Hi |

→ `01 06 D0 00 00 04 43 E9`

6. 전원 재투입 후 Max Coil Current 값을 다시 읽어 저장된 값을 확인한다.
7. 구동 중 전류가 설정된 제한값을 초과하지 않는지 확인한다.

### 판정 기준

- Max Coil Current [0xD13B] 파라미터 설정값이 정상 기록됨
- 전원 재투입 후에도 설정값 유지 (EEPROM 저장 확인)
- 구동 중 전류 제한 동작 정상 확인

**합격 조건**: 위 판정 조건 모두 만족

시험 항목 : [RS485 Modbus 기반 Max Coil Current 동적 제한 제어 및 EEPROM 기록 중단(Brownout) 자가 복구 통합 검증] 시험 목적 : Modbus FC 0x06 명령으로 Max Coil Current [0xD13B] 설정 및 EEPROM Save [0xD000] 명령이 정상 수행되는지 확인한다. 또한, 고부하 구동 중 동적 전류 제한(On-the-fly Clamping) 능력, 스펙 초과 값에 대한 Exception 방어, 그리고 Flash 메모리 기록 중 전원 차단 시 메모리 오염(Corruption) 방지 및 안전 복구 로직을 종합적으로 검증한다.

시험 방법 : <Phase 1. 초기 상태 연결 및 EEPROM Save 정상 동작 확인 (Base Setup & Persistence Test)>

RS485 인터페이스를 통해 드라이브(Node ID: 0x01)와 PC를 연결하고, 19200bps 통신을 개방한다.
현재 Max Coil Current [0xD13B] 값을 Read Holding Register (FC: 0x03)로 읽어온다. (01 03 D1 3B 00 01 14 30)
전류 제한값을 제품 정격의 50%(예: 0x01F4, 500mA)로 설정하기 위해 Write Single Register (FC: 0x06)를 송신한다.
EEPROM Save 명령(01 06 D0 00 00 04 43 E9)을 송신하여 비휘발성 메모리에 기록한다.
드라이브 주 전원(AC/DC)을 완전히 차단 후 10초 대기, 전원을 재투입하고 다시 [0xD13B]를 읽어 0x01F4 값이 유지되는지 확인한다.
<Phase 2. 고부하 구동 중 동적 제한(On-the-fly Clamping) 및 BVA 검증 (Dynamic Load Test)>

모터에 동력계(Dynamometer)나 물리적 브레이크를 걸어 높은 부하(정격의 80% 수준)를 인가한 상태에서 Run 명령을 내린다.
모터가 고부하 전류를 소모하며 회전하고 있을 때, 정지(Stop) 명령 없이 실시간으로 Max Coil Current [0xD13B] 값을 현재 소모 전류보다 현저히 낮은 20%(예: 0x00C8, 200mA) 경계값으로 하향 Write (FC: 0x06) 한다.
전류 프로브(Oscilloscope) 및 실시간 차트를 통해, 명령 수신 직후(10ms 이내) 모터의 상전류(Phase Current) 파형 윗단이 즉각적으로 잘리며(Clamping), 과전류 알람(OCP) 발생 없이 설정된 20% 레벨로 억제되는지 확인한다.
다시 제한값을 하드웨어 최대 허용치(100%)로 상향 Write 했을 때, 부드럽게 원래의 토크로 복귀하는지 관찰한다.
<Phase 3. 하드웨어 한계 초과 비정상 전류값 입력 예외 검증 (Negative Test)>

모터나 인버터 소자(IGBT/FET)를 태워먹을 수 있는 하드웨어 스펙 초과 값(예: 정격의 200% 또는 0xFFFF)을 Max Coil Current [0xD13B] 레지스터에 강제 Write 시도한다.
제어 불능을 유발할 수 있는 0 (0mA) 값을 강제 Write 시도한다.
통신 버퍼를 감시하며 펌웨어가 이 치명적인 파라미터 오염을 거부하는지 확인한다.
<Phase 4. EEPROM 기록 중 전원 차단(Blackout) 스트레스 및 자가 복구 검증 (Flash Recovery Test)>

Max Coil Current [0xD13B] 값을 새로운 정상 값(예: 정격 70%, 0x02BC)으로 Write 한다.
[극한 스트레스 주입] EEPROM Save 명령(01 06 D0 00 00 04 43 E9)을 전송함과 동시에(약 5~10ms 이내, 플래시 메모리 Write Cycle 진행 중) 드라이브의 주 전원을 강제로 절체(차단)시킨다.
10초 후 전원을 재투입하고 통신을 연결한다.
펌웨어가 메모리 손상(HardFault)으로 뻗지 않고 정상 부팅되는지, 그리고 파라미터 무결성 체크(Checksum/CRC) 로직이 동작하여 [0xD13B] 값이 깨진 쓰레기 값이 아닌 '이전 정상 값(50%)' 또는 '공장 초기화 값(Default)'으로 안전하게 자가 복구(Fallback) 되었는지 확인한다.
판정기준 : [Phase 1 판정 - 초기 연결 및 EEPROM]

Save 명령(0x0004 -> [0xD000]) 전송 시 정상 Echo 응답이 와야 하며, 전원 재투입 후에도 50%(0x01F4) 값이 정확히 파싱되어야 한다.
[Phase 2 판정 - 동적 전류 제한 (제어기 성능 핵심)]

구동 중 파라미터 변경 시 드라이브 리셋이나 Fault 발생이 없어야 한다.
[중요] 오실로스코프 측정 기준, 하향된 전류 제한값(20%)을 초과하는 Overshoot(전류 튐 현상)가 발생하여 하드웨어 OCP 트립(Trip)을 유발하면 즉시 불합격(Fail) 처리한다. 제어기 PI 출력이 설정된 Current Limit에 의해 즉각적이고 안정적으로 Saturation(포화) 되어야 합격이다.
[Phase 3 판정 - 예외 처리]

0xFFFF, 0 등 허용 범위 밖의 값 Write 시도 시, 드라이브는 해당 값을 메모리에 덮어쓰지 않고 즉시 Modbus Exception Code 0x03 (Illegal Data Value) 프레임으로 방어해야 한다.
[Phase 4 판정 - EEPROM 오염 방지 및 자가 복구 (신뢰성 핵심)]

Flash Write 중 전원이 날아가더라도, 재부팅 시 MCU가 데드락(벽돌 현상)에 빠지지 않고 무조건 정상적으로 통신 루프가 살아나야 한다. (소프트웨어 생존 보장)
메모리가 일부 깨졌더라도 펌웨어의 무결성 검증(CRC) 로직이 이를 감지하여, [0xD13B] 레지스터를 읽었을 때 0xABCD 같은 알 수 없는 값이 나오지 않고, 안전한 이전 값 또는 Default 값으로 자동 롤백(Rollback) 되어 모터 구동 시 폭주를 막아야 최종 합격이다.

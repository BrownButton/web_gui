# 구동동작 No. 02 - Set Value Source - Analog V

## 시험 항목

Set Value Source - Analog V

## 시험 목표

- Set Value Source [0xD101]을 AIN1(Analog V, 0x0000)으로 설정한 후 가변 전압 소스(0~10V)를 인가한다.
- 아날로그 입력 전압에 비례하여 팬 모터 속도/토크가 정상 제어됨을 확인한다.
- Actual Speed [0xD02D] 파라미터가 아날로그 전압 입력에 정상 추종함을 확인한다.

## 시험 기준

### 사전 조건

- RS485 인터페이스를 통해 드라이브와 호스트 PC 연결 완료
- 가변 전압 소스 준비 완료 (0~10V 범위, EC FAN Analog 입력 단자에 연결)
- Slave ID: 0x01 기준

### 시험 절차

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. Set Value Source를 Analog V 입력(AIN1, 0x0000)으로 설정한다.

**Set Value Source 쓰기 (AIN1 = 0x0000) — FC(06) Write Single Register**
- 레지스터: Set Value Source [0xD101]

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | 0xD1 | 0x01 | 0x00 | 0x00 | CRC Lo | CRC Hi |

4. 가변 전압 소스를 EC FAN Analog 입력 단자에 연결한다.
5. 다양한 전압값을 순서대로 인가한다.

| 입력 전압 | 기대 출력 |
|----------|----------|
| 0V | 0% (정지) |
| 1V | 최소 구동 수준 |
| 5V | 50% 출력 |
| 10V | 100% (최대 출력) |

6. 각 전압에 따라 팬 모터가 해당 속도 또는 토크로 구동되는지 확인한다.
7. Actual Speed [0xD02D] 파라미터를 읽어 속도 추종 여부를 확인한다.

**Actual Speed 읽기 명령 — FC(04) Read Input Register**

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x04 | 0xD0 | 0x2D | 0x00 | 0x01 | CRC Lo | CRC Hi |

→ `01 04 D0 2D 00 01 99 CB`

### 판정 기준

- 아날로그 입력 전압(0~10V)에 비례하여 팬 모터 속도/토크가 정상 제어됨
- Actual Speed 파라미터가 아날로그 전압 입력에 정상 추종

**합격 조건**: 위 판정 조건 모두 만족

시험 항목 : [RS485 Modbus 기반 아날로그(0-10V) 지령 선형 추종성 및 ADC 노이즈/단선(Floating) 연계 Failsafe 통합 검증] 시험 목적 : Set Value Source [0xD101]를 AIN1(0x0000)으로 설정 시, 0V에서 10V까지의 전압 입력에 따른 Actual Speed [0xD02D]의 선형적 추종성과 ADC 데드밴드(Deadband) 및 클램핑(Clamping) 처리 능력을 검증한다. 또한 쓰기 권한 침범 예외 처리, 스펙 초과 전압(Over-Voltage) 인가 시의 소프트웨어 방어 로직, 구동 중 신호선 단선 시 하드웨어 풀다운(Pull-down)에 의한 안전 정지(Failsafe) 복구 능력을 종합적으로 확인한다.

시험 방법 : <Phase 1. 초기 상태 설정 및 기본 아날로그 전압(0-10V) 선형 추종성 확인 (Base Setup & Linearity)>

RS485 인터페이스를 통해 드라이브(Node ID: 0x01)와 PC를 연결하고 19200bps 통신을 개방한다.
Set Value Source [0xD101] 레지스터에 Analog V 입력 모드(0x0000)를 Write Single Register (FC: 0x06) 명령으로 전송한다.
송신(TX) 프레임: 01 06 D1 01 00 00 D0 F6
프로그래머블 DC 파워 서플라이를 EC FAN의 아날로그 입력 단자(AIN1, GND)에 결선하고, 중간값인 5.0V를 인가한다.
모터가 기동하면 Actual Speed [0xD02D] 레지스터를 Read Input Register (FC: 0x04) 명령(01 04 D0 2D 00 01 99 CB)으로 100ms마다 연속 Polling 하여, 최대 RPM의 50% 속도를 오차 범위 내에서 안정적으로 유지하는지 확인한다.
<Phase 2. ADC 데드밴드(BVA) 정밀 조사 및 스펙 초과 전압(Over-limit) 스트레스 검증 (Boundary & Noise Test)>

[Deadband BVA] 전압을 0.0V에서 0.1V 단위로 미세하게 올린다. 노이즈 방어를 위한 하단 데드밴드 구간(예: 0.0V - 0.5V)에서는 모터가 절대 기동하지 않고 0 RPM을 유지하는지 확인한다.
[Clamping BVA] 전압을 9.5V에서 10.0V까지 올렸을 때 모터가 100% RPM에 도달하는지 확인하고, 상단 클램핑 구간을 검증한다.
[극한 스트레스 주입] 실수로 10V를 초과하는 과전압(예: 12.0V)이 아날로그 단자에 인가되었을 때, 펌웨어의 ADC 변수 타입이 오버플로우(Overflow)되어 속도가 0으로 곤두박질치거나 오작동하지 않고, **안전하게 100% RPM으로 소프트웨어 클램핑(Saturation)**을 유지하는지 확인한다.
<Phase 3. 지원 범위를 벗어난 비정상 모드 설정 및 파서 예외 검증 (Negative Test)>

Set Value Source [0xD101] 레지스터에 AIN1(0x0000)이나 규정된 모드가 아닌, 정의되지 않은 예외 값(예: 0x0009, 0xFFFF 등)을 강제로 Write (FC: 0x06) 시도한다.
예외 송신(TX) 프레임: 01 06 D1 01 FF FF [해당 CRC Hi/Lo]
통신 버퍼를 감시하며 파서(Parser)가 이 불법적인 소스 변경을 즉각 거부하는지 확인한다.
<Phase 4. 고속 구동 중 아날로그 제어선 단선(Wire Break) 및 자가 복구 검증 (Floating Signal & Failsafe Test)>

파워 서플라이를 10.0V로 설정하여 EC FAN을 100% 최대 속도로 맹렬히 회전시킨다.
[물리적 치명타 주입] 모터가 최대 관성으로 돌고 있는 도중, 아날로그 신호선(AIN1)을 드라이브 단자대에서 물리적으로 뽑아버린다(Wire Break 단선).
펌웨어 및 하드웨어 회로가 단선된 핀의 플로팅(Floating) 노이즈를 전압으로 오인하지 않고 내부 풀다운(Pull-down) 저항에 의해 즉각 0.0V로 인식하여, 모터를 0 RPM으로 안전하게 감속 정지(Failsafe) 시키는지 오실로스코프와 육안으로 철저히 관찰한다.
모터가 완전히 멈춘 후 통신 단절(Deadlock)이나 Fault가 없는지 [0xD02D]를 읽어 0 RPM을 확인하고, 다시 10.0V 신호선을 결선(Hot-Plug)한다.
드라이브가 즉시 10.0V 지령을 재인식하여 과전류(OCP) 알람 없이 부드럽게 100% 속도로 자가 복구되는지 최종 확인한다.
판정기준 : [Phase 1 판정 - 초기 설정 및 선형성]

[0xD101] 설정 시 정상 Echo 응답이 와야 하며, 1.0V 구간에서 10.0V 구간까지 Actual Speed 파라미터 값이 입력 전압에 비례하여 선형적으로 추종해야 한다. (속도 리플 오차 ±2% 이내 합격)
[Phase 2 판정 - 데드밴드 및 ADC 클램핑 (제어 안정성 핵심)]

데드밴드 구간(예: 0.5V 이하)에서 모터가 미세하게 진동(Jittering)하거나 고주파 소음을 내면 ADC 필터링 불량으로 즉시 **불합격(Fail)**이다.
12.0V 스펙 초과 인가 시, 데이터 오버플로우로 인해 모터가 역회전하거나 정지하는 등의 소프트웨어 버그가 없어야 하며 100% 속도를 안정적으로 억제(Clamp)해야 합격이다.
[Phase 3 판정 - 예외 처리]

0xFFFF 등 허용되지 않은 Source 값 입력 시, 드라이브는 즉각 Modbus Exception Code 0x03 (Illegal Data Value) 프레임(예: 01 86 03 02 61)을 반환하여 잘못된 제어권 변경을 차단해야 한다.
[Phase 4 판정 - 단선 방어 및 Failsafe (가장 중요한 하드웨어 안전성)]

케이블 단선 시, 내부 핀이 안테나처럼 노이즈를 빨아들여(Floating) 모터가 제멋대로 가감속을 반복하거나 폭주하면 **즉시 불합격(Fail)**이다.
시스템은 단선을 즉시 0V 지령으로 인식하여 부드럽게 정지(Stop)해야 하며, 신호 재연결 시 전원 리셋 없이 즉각적으로 정상 가동 상태로 복구되어야만 최종 합격 처리한다.

# 01. 프로젝트 개요

## 1. 무엇을 만들었나

**EC-FAN Modbus Dashboard**는 당사의 EC-FAN 모터 드라이브 제품을
PC 브라우저에서 실시간으로 모니터링하고 제어할 수 있는 웹 기반 도구입니다.

RS-485 버스에 연결된 최대 247개의 EC-FAN 슬레이브 장치를
Modbus RTU 프로토콜로 통신하며, 별도의 프로그램 설치 없이
Chrome 또는 Edge 브라우저만으로 사용할 수 있습니다.

---

## 2. 왜 웹으로 만들었나

### 기존 방식 (C# WinForm/WPF) 의 한계

기존에는 C#으로 Windows Forms 또는 WPF 앱을 만들어 사용해왔습니다.
이 방식의 단점은 다음과 같습니다:

| 문제 | 내용 |
|------|------|
| 설치 필요 | .NET 런타임 설치, 버전 관리 필요 |
| 배포 어려움 | 새 버전을 배포할 때마다 설치 파일 전달 |
| 플랫폼 종속 | Windows 전용 (Mac, Linux 사용 불가) |
| UI 개발 비용 | WPF XAML 러닝 커브가 높음 |

### 웹 방식의 장점

| 장점 | 내용 |
|------|------|
| 설치 불필요 | 브라우저만 있으면 즉시 실행 |
| 즉각적인 배포 | 파일 교체만으로 모든 사용자에게 반영 |
| 풍부한 UI | 웹 기술로 고품질 인터페이스 구현 |
| AI 활용 최적화 | 텍스트 기반 코드여서 AI와의 협업에 최적 |

### 웹 방식의 단점

| 단점 | 내용 | 비고 |
|------|------|------|
| **소스코드 완전 공개** | HTML/JS/CSS는 F12(개발자 도구)로 누구나 원문 그대로 열람 가능. C#처럼 컴파일된 바이너리가 없음 | 이 앱의 핵심은 Modbus 표준 통신이며, 실제 지적재산(제어 알고리즘 등)은 EC-FAN 펌웨어 내부에 있음. 실질적 문제 없음 |
| **브라우저 종속** | Chrome/Edge만 지원. Firefox, Safari 사용 불가. 브라우저 버전에 따라 동작이 달라질 수 있음 | 사용 전 Chrome/Edge 설치 필요 |
| **Web Serial API 안정성** | 비교적 신규 API (2021년 정식 도입). 향후 브라우저 정책 변경으로 동작이 바뀔 가능성 존재 | 장기 유지보수 시 주의 필요 |
| **개발자 도구로 내부 조작 가능** | 브라우저 콘솔에서 `window.dashboard.writeRegister(...)` 등 내부 함수를 직접 호출 가능 | 사내 사용 환경에서는 실질적 위험 낮음 |

> **요약**: 사내 생산·검사 툴 용도에서는 단점들의 실질적 영향이 거의 없음. 가장 주의할 부분은 브라우저 종속성(Chrome/Edge 필수)과 Web Serial API의 장기 지원 여부.

### 왜 웹인데 시리얼 통신이 가능한가?

일반적으로 웹은 하드웨어 접근이 불가능합니다.
하지만 **Web Serial API**라는 최신 브라우저 기능을 사용하면
브라우저에서 직접 COM 포트(RS-485, USB-시리얼 등)에 접근할 수 있습니다.

> 자세한 내용은 [05_WebSerial_통신.md](05_WebSerial_통신.md) 참조

## 3. 주요 기능

### 대시보드 (Dashboard)

- 연결된 모든 EC-FAN 장치를 카드 형태로 표시
- 각 장치의 실시간 속도, 상태, 통신 성공률 표시
- 전체 장치 일괄 제어 (속도 설정, 방향 전환 등)
- 개별 장치 ⚙ 버튼으로 상세 설정 팝업 열기
- 장치 자동 스캔 및 Slave ID 자동 할당

### Device Setup (장치 설정)

- 개별 장치의 파라미터 읽기/쓰기
- Configuration 탭: Fan Address, 운전 방향, 운전 모드, 최대 전류 설정
- Parameters 탭: 모든 레지스터 값 일괄 읽기
- Manufacture 탭: 제조 검증, 하드웨어 테스트, 튜닝, 오프셋, 시리얼 번호

### Modbus (수동 통신)

- 기능 코드 선택 (FC01~FC16)
- 레지스터 주소 및 데이터 직접 입력
- 요청/응답 프레임을 HEX로 표시
- 통신 모니터 (수신/송신 로그, HEX/DEC 보기)

### Update (펌웨어 업데이트)

- 바이너리 파일 선택 후 장치로 전송
- 전송 진행률 표시
- 업데이트 완료 후 재부팅 확인

### Chart (실시간 차트)

- 4채널 독립 실시간 데이터 플롯
- 마우스 휠 줌, 드래그 팬
- 트리거 모드 (특정 값에서 캡처)
- 커서로 정확한 값 읽기
- 1,000 샘플 버퍼

### Settings (앱 설정)

- 폴링 간격 조정
- 자동 스캔 범위 설정
- 시뮬레이터 모드 (실제 하드웨어 없이 테스트)
- 통계 초기화

## 4. 시스템 구성

```
┌─────────────────────────────────────────────────────┐
│                    PC (브라우저)                     │
│                                                     │
│  ┌──────────────────────────────────────────────┐  │
│  │       EC-FAN Modbus Dashboard (웹앱)          │  │
│  │   Chrome / Edge 브라우저에서 실행              │  │
│  └──────────────────┬───────────────────────────┘  │
│                     │ Web Serial API                │
│                     │ (USB COM 포트)                │
└─────────────────────┼───────────────────────────────┘
                      │
               [USB-RS485 변환기]
                      │
              ╔═══════╧═══════╗
              ║   RS-485 버스  ║
              ╠═══════════════╣
              ║  EC-FAN #1    ║ (Slave ID 1)
              ║  EC-FAN #2    ║ (Slave ID 2)
              ║  EC-FAN #...  ║
              ║  EC-FAN #247  ║ (Slave ID 247 까지)
              ╚═══════════════╝
```

## 5. 지원 환경

| 항목 | 요구사항 |
|------|----------|
| 브라우저 | Chrome 89+ 또는 Edge 89+ |
| OS | Windows / macOS / Linux / ChromeOS + Android Chrome (USB OTG 환경). iOS 미지원 |
| 연결 | USB-RS485 변환기 또는 COM 포트 |
| 설치 | 없음 (index.html 파일을 브라우저로 열기만 하면 됨) |

> **주의**: Firefox, Safari는 Web Serial API를 지원하지 않습니다.

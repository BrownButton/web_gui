# 3-01. Software Reset

## 시험 항목

Modbus 명령을 통해 드라이브 소프트웨어 리셋(Software Reset)이 정상 동작하는지 확인하는 시험이다.

## 시험 목적

Modbus 명령을 통해 드라이브 소프트웨어 리셋(Software Reset)이 정상 동작하는지 확인한다.

## 시험 방법

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. 드라이브의 정상 동작 상태를 확인한다. (파라미터 읽기 가능 여부 확인)
4. Software Reset 명령을 아래 프레임을 참조하여 입력한다.

**SW Reset 명령 (0xD000 레지스터에 Reset 코드 쓰기)**

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | 0xD0 | 0x00 | Reset Code Hi | Reset Code Lo | CRC Hi | CRC Lo |

5. 리셋 명령 전송 후 드라이브가 재시작되는지 확인한다.
   - 통신 응답이 일시적으로 끊겼다가 재연결되는 현상 확인
6. 재시작 후 파라미터를 읽어 정상 동작 상태인지 확인한다.

## 판정기준

- SW Reset 명령 전송 후 드라이브가 정상적으로 재시작됨
- 재시작 후 통신 및 파라미터 읽기가 정상 동작

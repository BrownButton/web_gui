# 3-13. OS 다운로드 중 분리 예외처리

## 시험 항목

OS 다운로드 진행 중 통신 연결이 끊겼을 때 재연결 후 OS 재다운로드가 가능한지 확인하는 시험이다.

## 시험 목적

OS 다운로드 진행 중 통신 연결이 끊겼을 때 재 연결 시 OS 재 다운로드가 가능한지 확인한다.

## 시험 방법

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. EC FAN OS 업데이트 툴을 실행한다.
4. 통신 사양을 설정한 후 Connect 버튼을 누른다.
   - (Baudrate 19,200bps, Even Parity 조건은 공장 출하 상태에서의 초기 값이며, 이 값이 팬 드라이브 설정과 다를 경우 접속이 되지 않으므로 유의)
5. 다운로드할 F/W 파일을 선택한 후 Forced Download 체크박스를 체크한다.
6. Start 버튼을 눌러 다운로드를 진행한다.
7. **다운로드 중 RS485 케이블을 분리한다.**
   - 다운로드 진행 중간(예: 50% 진행 시점)에 케이블 분리
8. 분리된 RS485 케이블을 다시 접속한 후 OS 업데이트를 다시 진행한다.
9. 전원 재 투입 후 통신 재 접속하여 정상적으로 업데이트 되었는지 확인한다.

## 판정기준

- OS 재 다운로드하여 정상적으로 업데이트 되었을 경우 합격
  - (Main OS의 경우 LED 점멸 주기 변화를 통해 업데이트 확인)

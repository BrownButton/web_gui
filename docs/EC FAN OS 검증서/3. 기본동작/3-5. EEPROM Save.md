# 3-05. EEPROM Save

## 시험 항목

파라미터 변경 후 EEPROM Save 명령 실행 여부에 따라 전원 재투입 시 파라미터가 유지 또는 초기화되는지 확인하는 시험이다.

## 시험 목적

파라미터 변경 후 EEPROM Save 명령을 실행하면 전원 재투입 후에도 변경된 파라미터가 유지되는지 확인한다.

## 시험 방법

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. 임의의 파라미터 값을 변경한다. (예: Set Point [0xD001], Running Direction [0xD102] 등)
4. 변경된 값을 FC(03)으로 읽어 변경되었음을 확인한다.

### EEPROM Save 없이 전원 재투입

5. EEPROM Save 명령 없이 전원을 OFF/ON한다.
6. 전원 재투입 후 변경한 파라미터를 읽어 값이 초기화되었는지 확인한다.
   - (EEPROM Save 없이 전원 재투입 시 이전 저장값으로 복구되어야 함)

### EEPROM Save 후 전원 재투입

7. 파라미터 값을 다시 변경한다.
8. EEPROM Save 명령을 전송한다.

**EEPROM Save 명령 (0xD000 레지스터에 Save 코드 쓰기)**

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | 0xD0 | 0x00 | EEPROM Save Code Hi | EEPROM Save Code Lo | CRC Hi | CRC Lo |

9. 전원을 OFF/ON한다.
10. 전원 재투입 후 변경한 파라미터를 읽어 EEPROM에 저장된 값이 유지되는지 확인한다.

## 판정기준

- EEPROM Save 명령 없이 전원 재투입 시: 이전 저장값으로 복구
- EEPROM Save 후 전원 재투입 시: 변경된 값이 유지됨

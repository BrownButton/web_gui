# 1-02. Broadcast 동작 시험

## 시험 항목

Node ID 0 (Broadcast)으로 명령 전송 시 RS485 버스에 연결된 모든 드라이브에 동시 적용되는지 확인하는 시험이다.

## 시험 목적

Node ID 0 (Broadcast)으로 명령 전송 시 RS485 버스에 연결된 모든 드라이브에 동시 적용되는지 확인한다.

## 시험 방법

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. EC FAN 2대를 RS485 버스에 병렬 연결한다.
4. 각 드라이브에 서로 다른 Node ID를 설정한다. (예: Node 1, Node 2)
   - Node ID 설정 방법은 RS485 No. 1-3 시험 참조
5. Broadcast ID(0x00)로 Set Point 명령을 전송한다.

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|-----|------|------|------|------|------|------|------|
| 내용 | 0x00 | 0x06 | 0xD0 | 0x01 | 0x19 | 0x9A | CRC Hi | CRC Lo |

- Broadcast ID: 0x00 (응답 없음, 단방향 전송)
- FC 0x06: Write Single Register
- 주소 0xD001: Set Point

6. 연결된 모든 드라이브가 명령에 반응하는지 (모터 구동 등) 확인한다.
7. 각 드라이브의 Set Point 파라미터 값을 개별적으로 읽어 동일하게 설정되었는지 확인한다.

## 판정기준

- Broadcast ID(0x00)로 전송한 명령이 버스에 연결된 모든 드라이브에 동시 적용

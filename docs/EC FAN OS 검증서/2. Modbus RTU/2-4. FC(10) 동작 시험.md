# 2-04. FC(10) 동작 시험

## 시험 항목

Modbus RTU Write Multiple Registers [0x10] 명령 입력 시 복수의 레지스터가 정상적으로 기록되는지 확인하는 시험이다.

## 시험 목적

Modbus RTU Write Multiple Registers [0x10] 명령 입력 시 복수의 레지스터가 정상적으로 기록되는지 확인한다.

## 시험 방법

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. Write Multiple Registers [0x10] 명령으로 연속된 복수 레지스터에 값을 기록한다.

**요청 프레임 예시 (Operating Mode [0xD106]과 Running Direction [0xD102] 2개 연속 쓰기)**

| 필드 | Node ID | Function | Start Addr Hi | Start Addr Lo | Qty Hi | Qty Lo | Byte Count | Data... | CRC |
|------|---------|----------|--------------|--------------|--------|--------|------------|---------|-----|
| 내용 | 0x01 | 0x10 | 0xD1 | 0x02 | 0x00 | 0x02 | 0x04 | 데이터 4바이트 | - |

4. 명령 코드 송신 후, 수신되는 응답 프레임을 확인한다.

**응답 프레임**

| 필드 | Node ID | Function | Start Addr Hi | Start Addr Lo | Qty Hi | Qty Lo | CRC |
|------|---------|----------|--------------|--------------|--------|--------|-----|
| 내용 | 0x01 | 0x10 | 0xD1 | 0x02 | 0x00 | 0x02 | - |

5. FC(03)으로 기록된 레지스터 값들을 읽어 정상 기록되었는지 확인한다.

6. 잘못된 값 송신 시 에러코드를 확인한다.

**에러 응답 프레임**

| 바이트 | 0 | 1 | 2 | 3 | 4 |
|--------|------|------|---------|---|---|
| 내용 | 0x01 | 0x90 | 0x01~0x06 | - | - |

## 판정기준

- 복수 레지스터 쓰기 명령에 대한 응답이 정상으로 나타나면 합격
- FC(03) 읽기를 통해 기록된 값이 요청값과 일치함을 확인

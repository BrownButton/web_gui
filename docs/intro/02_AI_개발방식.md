# 02. AI 개발 방식 (Claude Code)

## 1. Claude Code란?

**Claude Code**는 Anthropic(앤트로픽)이 만든 AI 코딩 어시스턴트입니다.
VS Code 편집기의 확장 프로그램 또는 터미널 CLI로 사용할 수 있으며,
개발자가 자연어(한국어 포함)로 요구사항을 설명하면 AI가 코드를 생성, 수정, 설명합니다.

- **개발사**: Anthropic (구글, 아마존 투자, OpenAI 창립 멤버들이 설립)
- **모델**: Claude Sonnet / Opus (GPT-4급 대형 언어 모델)
- **사용 방식**: VS Code 확장 + 대화형 인터페이스

> Anthropic 공식 사이트: https://www.anthropic.com
> Claude Code 소개: https://claude.ai/claude-code

---

## 2. 기존 개발 방식과의 비교

### 기존 방식 (직접 코딩)

```
개발자 → 요구사항 분석 → 알고리즘 설계 → 직접 코드 작성 → 디버깅 → 완료
         (수시간~수일)    (머릿속으로)    (타이핑)         (반복)
```

### AI 협업 방식

```
개발자 → 자연어로 요구사항 설명 → AI가 코드 생성 → 개발자가 검토/수정 → 완료
         ("폴링 중 Setpoint를      (수초~수분)       (코드 리뷰)
          변경할 수 있게 해줘")
```

**핵심 차이점**: 개발자는 "무엇을" 원하는지에 집중하고,
"어떻게" 구현할지는 AI와 함께 결정합니다.

## 3. 실제 개발 흐름

### 단계 1: 프로젝트 컨텍스트 설정 (CLAUDE.md)

프로젝트 루트에 `CLAUDE.md` 파일을 만들어 AI에게 프로젝트 규칙을 알려줍니다.

```markdown
# CLAUDE.md 예시 (실제 파일 발췌)

## ⚠️ 485 버스 접근 규칙 (가장 중요)

### 절대 금지: `this.writer.write(frame)` 직접 호출
새 기능 추가 시 반드시 아래 안전한 경로를 통해야 함.

| 용도 | 사용할 함수 |
|------|------------|
| 레지스터 쓰기 | writeRegister(slaveId, address, value) |
| 홀딩 레지스터 읽기 | readRegisterWithTimeout(slaveId, address) |
```

이렇게 하면 AI는 매번 대화마다 이 규칙을 기억하고 코드를 작성합니다.
개발팀의 코딩 컨벤션을 AI에게 "가르치는" 역할입니다.

### 단계 2: 요구사항 자연어 지시

개발자가 채팅창에 요구사항을 입력합니다:

```
사용자: "Dashboard에서 팬 속도를 슬라이더로 조절할 수 있게 해줘.
        슬라이더 드래그 중에는 실시간으로 값이 표시되고,
        드래그가 끝났을 때만 실제로 전송해줘.
        RS-485 버스 충돌이 안 나게 해줘."
```

### 단계 3: AI가 코드 생성

AI는 기존 코드를 분석하고, CLAUDE.md 규칙을 따르며, 최적의 코드를 제안합니다.

```javascript
// AI가 생성한 코드 예시
slider.addEventListener('input', (e) => {
    valueDisplay.textContent = e.target.value + ' RPM';  // 실시간 표시만
});

slider.addEventListener('change', (e) => {
    const rpm = parseInt(e.target.value);
    const raw = Math.round(rpm / maxSpeed * 64000);
    writeRegister(device.slaveId, 0xD001, raw);  // 드래그 끝날 때만 전송
    // ✓ writeRegister를 사용하므로 버스 충돌 없음
});
```

### 단계 4: 개발자 검토 및 승인

AI가 제안한 코드를 개발자가 검토합니다:
- 로직이 맞는지 확인
- 버스 충돌 규칙을 지켰는지 확인
- 사이드 이펙트가 없는지 확인

문제가 없으면 적용, 수정이 필요하면 AI에게 피드백을 줍니다.

### 단계 5: 반복 개선

실제 하드웨어에서 테스트하며 문제가 발견되면 AI에게 다시 설명하고 수정합니다.

## 4. 디렉토리 권한 설정

AI가 코드를 작성하거나 파일을 편집하려면 해당 디렉토리에 대한 **접근 권한**이 필요합니다.
Claude Code는 `.claude/settings.local.json` 파일로 권한을 관리합니다.

### 권한 파일 위치

```
c:\github\web_gui\
└── .claude\
    └── settings.local.json   ← 권한 설정 파일
```

### 권한 파일 내용 예시

```json
{
  "permissions": {
    "allow": [
      "Bash(git status)",
      "Bash(git diff*)",
      "Bash(git log*)",
      "Bash(mkdir*)"
    ],
    "deny": []
  }
}
```

### 권한의 의미

| 권한 | 설명 |
|------|------|
| `allow` | AI가 실행할 수 있는 명령어 목록 |
| `deny` | AI가 실행할 수 없도록 차단된 명령어 |

### 사용자 승인 방식

`settings.local.json`에 없는 새로운 작업을 AI가 시도할 때마다
**화면에 팝업이 표시되어 사용자가 허용/거부를 선택**합니다.

```
┌─────────────────────────────────────────────┐
│ Claude가 다음 명령을 실행하려 합니다:         │
│                                             │
│  $ git add docs/README.md                  │
│                                             │
│  [허용]  [거부]  [항상 허용]                 │
└─────────────────────────────────────────────┘
```

즉, AI는 사용자가 명시적으로 허용한 작업만 수행할 수 있습니다.
**파일 삭제, 외부 서버 접근 등 위험한 작업은 사용자가 거부할 수 있습니다.**

## 5. AI 코드 품질 보장 방법

### CLAUDE.md 규칙화

핵심 아키텍처 규칙을 CLAUDE.md에 명문화합니다.
AI는 항상 이 파일을 먼저 읽고 코드를 작성합니다.

예: "절대 `this.writer.write()` 직접 호출 금지"
→ AI가 이 규칙을 어기면 개발자가 즉시 인지하고 수정을 요청할 수 있습니다.

### 개발자 최종 검토

AI가 생성한 코드는 **반드시 개발자가 검토하고 승인**합니다.
AI는 코드를 "제안"하는 역할이며, 최종 결정은 개발자가 합니다.

### Git으로 이력 관리

모든 변경사항은 Git으로 커밋되어 이력이 남습니다.
문제가 발생하면 이전 버전으로 되돌릴 수 있습니다.

```
git log --oneline
b842050 Update app.js   ← 최신 커밋
5a8adc1 Update app.js
e2fb351 Update parameters.csv
```

### 테스트 프레임워크

프로젝트에는 `os-test-manager.js`라는 자동화 테스트 프레임워크가 포함되어 있어
RS-485 통신 및 Modbus 프로토콜을 자동으로 검증합니다.

## 6. 장점과 주의사항

### 장점

| 장점 | 설명 |
|------|------|
| 빠른 개발 속도 | 반복적인 코드 작성 시간 단축 |
| 낮은 진입 장벽 | 웹 기술을 깊이 몰라도 기능 추가 가능 |
| 코드 품질 | AI가 최신 best practice에 따라 코드 작성 |
| 문서화 | AI가 코드 설명 및 주석 작성 도움 |
| 디버깅 | 에러 메시지를 AI에게 붙여넣으면 원인 분석 |

### 주의사항

| 주의 | 설명 |
|------|------|
| 도메인 지식 필수 | AI가 업무 도메인(Modbus, 모터 제어)을 모르면 잘못된 코드 생성 가능. 개발자가 검토 필수 |
| 사용 비용 | API 사용량에 따라 요금 발생 (월정액 또는 사용량 기반) |
| 인터넷 연결 | AI 서버와 통신이 필요하므로 인터넷 연결 필수 |
| 코드 보안 | 코드가 AI 서버로 전송됨. 기밀 알고리즘은 주의 필요 |

## 7. 이 프로젝트에서 AI가 한 일

이 프로젝트의 거의 모든 코드는 AI와의 협업으로 작성되었습니다:

- **UI 설계**: 대시보드 레이아웃, 카드 컴포넌트, 모달 팝업
- **통신 구현**: Web Serial API 연결, Modbus RTU 프레임 생성/파싱
- **버스 안전 구조**: Command Queue 패턴, 폴링/쓰기 동시성 처리
- **실시간 차트**: Canvas API 기반 4채널 플로터
- **테스트 프레임워크**: 자동화 검증 시스템
- **시뮬레이터**: 하드웨어 없이 테스트 가능한 가상 장치

개발자의 역할:
- Modbus 프로토콜 및 EC-FAN 장치 스펙 제공
- AI가 생성한 코드 검토 및 피드백
- 실제 하드웨어 테스트 및 버그 보고
- CLAUDE.md로 개발 규칙 정의

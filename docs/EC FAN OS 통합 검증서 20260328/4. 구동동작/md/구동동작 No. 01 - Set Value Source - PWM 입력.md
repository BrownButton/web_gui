# 구동동작 No. 01 - Set Value Source - PWM 입력

## 시험 항목

Set Value Source - PWM 입력

## 시험 목표

- Set Value Source [0xD101]을 PWM 모드(0x0003)로 설정한 후 PWM 신호 발생기에서 다양한 듀티비를 인가한다.
- PWM 듀티비에 비례하여 팬 모터 속도/토크가 정상 제어됨을 확인한다.
- Actual Speed [0xD02D] 파라미터가 PWM 입력에 정상 추종함을 확인한다.

## 시험 기준

### 사전 조건

- RS485 인터페이스를 통해 드라이브와 호스트 PC 연결 완료
- PWM 신호 발생기 준비 완료 (EC FAN PWM 입력 단자에 연결)
- Slave ID: 0x01 기준

### 시험 절차

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. Set Value Source를 PWM 입력(0x0003)으로 설정한다.

**Set Value Source 쓰기 (PWM = 0x0003) — FC(06) Write Single Register**
- 레지스터: Set Value Source [0xD101]

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | 0xD1 | 0x01 | 0x00 | 0x03 | CRC Lo | CRC Hi |

4. PWM 신호 발생기를 EC FAN PWM 입력 단자에 연결한다.
5. 다양한 듀티비의 PWM 신호를 순서대로 인가한다.

| PWM 듀티비 | 기대 출력 |
|-----------|----------|
| 10% | 최소 구동 수준 |
| 50% | 중간 속도/토크 |
| 90% | 최대 구동 수준 |

6. 각 듀티비에 따라 팬 모터가 해당 속도 또는 토크로 구동되는지 확인한다.
7. Actual Speed [0xD02D] 파라미터를 읽어 속도 추종 여부를 확인한다.

**Actual Speed 읽기 명령 — FC(04) Read Input Register**

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x04 | 0xD0 | 0x2D | 0x00 | 0x01 | CRC Lo | CRC Hi |

→ `01 04 D0 2D 00 01 99 CB`

### 판정 기준

- PWM 듀티비에 비례하여 팬 모터 속도/토크가 정상 제어됨
- Actual Speed 파라미터가 PWM 입력에 정상 추종

시험 항목 : [RS485 Modbus 기반 외부 PWM 지령(Set Value) 동적 추종성 및 제어선 단선(Signal Loss) 연계 Failsafe 안전 복구 통합 검증] 시험 목적 : Set Value Source [0xD101]를 PWM 모드(0x0003)로 설정 시, 외부 PWM 듀티비 입력에 따른 Actual Speed [0xD02D]의 선형적 추종성과 데드밴드(Deadband) 경계값을 검증한다. 또한 쓰기 권한/주파수 범위 이탈에 대한 예외 처리와, 구동 중 PWM 신호선 단선(Wire Break) 시 시스템이 물리적 데미지나 폭주 없이 설계된 Failsafe(안전 정지) 상태로 진입 및 복구되는지 종합적으로 확인한다.

시험 방법 : <Phase 1. 초기 상태 설정 및 기본 PWM 듀티비 선형 추종성 확인 (Base Setup & Happy Path)>

RS485 인터페이스를 통해 드라이브(Node ID: 0x01)와 PC를 연결하고 통신(19200bps, Even)을 개방한다.
Set Value Source [0xD101] 레지스터에 PWM 입력 모드(0x0003)를 Write Single Register (FC: 0x06) 명령으로 설정한다.
송신(TX) 프레임: 01 06 D1 01 00 03 9C F7 (맞는 CRC 적용)
정밀 함수 발생기(Function Generator)를 EC FAN의 PWM 입력 단자(예: 10V PWM)에 결선하고, 표준 주파수인 1kHz, 듀티비 50% 신호를 인가한다.
모터가 기동하여 회전하면, Actual Speed [0xD02D] 레지스터를 Read Input Register (FC: 0x04) 명령(01 04 D0 2D 00 01 99 CB)으로 읽어 현재 모터의 RPM이 최대 RPM의 약 50% 수준을 정확히 유지하는지 확인한다.
<Phase 2. PWM 듀티비 BVA(경계값) 전수 조사 및 주파수 한계 스트레스 검증 (Boundary & Frequency Test)>

[Duty BVA] 함수 발생기의 듀티비를 0%에서 100%까지 1% 단위로 서서히 올리며 [0xD02D]를 100ms 주기로 연속 Polling(차트 기록)한다.
펌웨어에 설정된 하단 데드밴드(예: 05% 구간)에서는 모터가 확실히 정지(0 RPM)를 유지하는지, 상단(예: 95100%)에서는 최대 RPM으로 클램핑(Clamping)되어 흔들리지 않는지 확인한다.
[Frequency Stress] 듀티비를 50%로 고정한 상태에서, PWM 주파수를 스펙 외곽인 50Hz(초저주파)와 10kHz(초고주파 노이즈 환경)로 급격히 변경한다.
펌웨어의 PWM Capture 타이머(Input Capture) 로직이 비정상 주파수 유입 시 속도가 출렁이거나 제어기가 뻗지 않고, 내부 필터링을 통해 속도를 유지하거나 에러로 방어하는지 모니터링한다.
<Phase 3. 지원 범위를 벗어난 비정상 모드 설정 및 예외 검증 (Negative Test)>

Set Value Source [0xD101] 레지스터에 규정된 모드(예: 0~3)가 아닌, 정의되지 않은 쓰레기 값(예: 0x0004, 0xFFFF)을 강제로 Write (FC: 0x06) 전송 시도한다.
예외 송신(TX) 프레임: 01 06 D1 01 FF FF [맞는 CRC Hi/Lo]
통신 버퍼를 감시하며 펌웨어가 이 불법적인 소스 변경 명령을 파서 단에서 거부하는지 확인한다.
<Phase 4. 고속 구동 중 PWM 제어선 단선(Wire Break) 및 자가 복구 검증 (Signal Loss & Failsafe Recovery Test)>

함수 발생기를 1kHz, 80% 듀티비로 설정하여 EC FAN을 고속(예: 2500 RPM)으로 회전시킨다.
[극한 물리 스트레스] 모터가 맹렬히 돌고 있는 도중, 함수 발생기와 드라이브를 연결하는 PWM 신호선(Signal Wire)을 손으로 홱 뽑아버린다(Wire Break 물리적 단선).
펌웨어가 신호 상실을 즉시 감지하고 내부 풀다운(또는 풀업) 하드웨어 설계에 맞추어 모터를 즉각 0 RPM으로 안전하게 감속 정지(Failsafe) 시키는지 오실로스코프와 육안으로 철저히 확인한다. (단선 시 쓰레기 노이즈를 듀티비로 오인하여 모터가 급발진/진동하면 치명적 결함임)
모터가 완전히 정지한 후, 다시 PWM 신호선을 결선(Hot-Plug)한다.
드라이브가 하드웨어 Fault(전원 리셋) 상태에 빠지지 않고, 즉시 80% 지령을 재인식하여 인버터 과전류(OCP) 없이 부드러운 가속 곡선(Soft-start)을 타며 2500 RPM으로 자가 복구되는지 최종 확인한다.
판정기준 : [Phase 1 판정 - 초기 설정 및 응답성]

FC 0x06 Write 시 정상적인 Echo 응답이 수신되어야 하며, 인가된 50% 듀티비에 대한 Actual Speed 파라미터 값(RPM)의 추종 오차가 ±2% 이내여야 합격 처리한다.
[Phase 2 판정 - 경계값 및 필터링 성능 (제어기 성능 핵심)]

데드밴드 구간(예: 0~5%) 진입 시 모터는 미세하게 떨거나(Jitter) 고주파 소음을 내지 않고 완벽한 PWM 출력 차단(0 RPM) 상태를 유지해야 한다.
50Hz 또는 10kHz 비정상 주파수 인가 시 MCU 타이머 오버플로우로 인해 모터가 급발진(100% 출력)하는 등의 소프트웨어 버그가 없어야 합격이다.
[Phase 3 판정 - 예외 처리]

0xFFFF 등 허용되지 않은 Source 값 입력 시, 드라이브는 즉각 파라미터 갱신을 거부하고 Modbus Exception Code 0x03 (Illegal Data Value) 프레임(예: 01 86 03 02 61)을 반환하여 파라미터 오염을 방어해야 한다.
[Phase 4 판정 - 단선 방어 및 Failsafe (가장 중요한 하드웨어 안전성)]

PWM 신호선이 끊겼을 때 내부 핀이 플로팅(Floating) 되어 노이즈를 듀티비로 잘못 읽고 **모터가 미친 듯이 떨리거나 원치 않는 급가속을 하면 즉시 불합격(Fail)**이다.
시스템은 단선 상태를 명확히 0% 듀티(또는 설계된 Failsafe 속도)로 인식하여 안전하게 정지해야 하며, 신호선 재연결 시 전원 리셋 없이 즉각적이고 부드럽게 기존 속도로 재가동되어야만 최종 합격 처리한다.

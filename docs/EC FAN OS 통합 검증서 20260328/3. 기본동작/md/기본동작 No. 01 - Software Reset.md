# 기본동작 No. 01 - Software Reset

## 시험 항목

Software Reset

## 시험 목표

- Modbus FC(06) 명령으로 Reset 레지스터 [0xD000]에 Software Reset 코드(0x0001)를 전송하면 드라이브가 정상적으로 재시작됨을 확인한다.
- 재시작 후 통신 및 파라미터 읽기가 정상 동작함을 확인한다.

## 시험 기준

### 사전 조건

- RS485 인터페이스를 통해 드라이브와 호스트 PC 연결 완료
- 드라이브 정상 동작 상태 확인 (파라미터 읽기 가능)
- Slave ID: 0x01 기준 (실제 설정값에 맞게 변경)

### 시험 절차

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. 드라이브의 정상 동작 상태를 확인한다. (파라미터 읽기 가능 여부 확인)
4. Software Reset 명령을 아래 프레임을 참조하여 전송한다.

**SW Reset 명령 — FC(06) Write Single Register**
- 레지스터: Reset [0xD000]
- 값: 0x0001 (Software Reset)

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | 0xD0 | 0x00 | 0x00 | 0x01 | CRC Lo | CRC Hi |

→ `01 06 D0 00 00 01 42 28`

5. 리셋 명령 전송 후 드라이브가 재시작되는지 확인한다.
   - 통신 응답이 일시적으로 끊겼다가 재연결되는 현상 확인
6. 재시작 후 파라미터를 읽어 정상 동작 상태인지 확인한다.

**Motor Status 읽기로 정상 복귀 확인 — FC(04) Read Input Register**
- 레지스터: Motor Status [0xD011]

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x04 | 0xD0 | 0x11 | 0x00 | 0x01 | 0x59 | 0x0F |

→ `01 04 D0 11 00 01 59 0F`

### 판정 기준

- SW Reset 명령 전송 후 드라이브가 정상적으로 재시작됨
- 재시작 후 통신 및 파라미터 읽기가 정상 동작

시험 항목 : [RS485 Modbus 기반 펌웨어 Software Reset (FC: 0x06) 동작 무결성 및 구동 중 강제 리셋 예외/복구 통합 검증]

시험 목적 : Modbus FC 0x06 명령으로 Reset 레지스터 [0xD000]에 Software Reset 코드(0x0001)를 전송했을 때 정상적인 ACK 응답 후 재부팅이 수행되는지, 비정상 리셋 코드 입력 시의 방어 로직, 재부팅 소요 시간(Boot Time), 그리고 모터 구동 중 강제 리셋 시 PWM 출력을 즉각 차단하고 안전한 상태(Stop/Ready)로 복구되는지 종합적으로 검증한다.

시험 방법 : <Phase 1. 초기 상태 연결 및 비휘발성 파라미터 백업 확인 (Base Setup)>

RS485 인터페이스를 통해 드라이브(Node ID: 0x01)와 PC를 연결하고, 통신 프로그램(19200bps, Even, 8, 1)을 오픈한다.
재부팅 후 데이터 유지 여부를 확인하기 위해, 임의의 제어 파라미터(예: Target RPM)를 특정 값(예: 1500)으로 Write하고 EEPROM에 Save 명령을 내린다.
Motor Status [0xD011] 레지스터를 Read Input Register (FC: 0x04) 명령(01 04 D0 11 00 01 59 0F)으로 읽어, 현재 드라이브가 정상 대기(Ready) 상태인지 확인한다.
<Phase 2. Software Reset 명령 인가 및 부팅/복구 타이밍 검증 (Positive Test)>

Reset [0xD000] 레지스터에 Software Reset 코드(0x0001)를 Write Single Register (FC: 0x06) 명령으로 전송한다.
송신(TX) 프레임: 01 06 D0 00 00 01 42 28
[핵심 측정] 전송 직후 수신(RX) 버퍼를 모니터링하여, 펌웨어가 리셋(MCU 재시작) 동작에 들어가기 직전에 마스터에게 동일한 Echo 프레임을 정상 응답(ACK)하는지 확인한다.
응답 수신 직후부터 100ms 주기로 Motor Status [0xD011] 읽기 명령(01 04 D0 11 00 01 59 0F)을 연속 Polling 전송하며, 통신이 단절되는 시간과 다시 정상 응답(RX)이 돌아오기까지의 '부팅 소요 시간(Boot Time)'을 스톱워치/로그로 측정한다.
재부팅 완료 후 Phase 1에서 설정했던 Target RPM(1500) 값이 초기화되지 않고 정상 보존되어 있는지 확인한다.
<Phase 3. 정의되지 않은 비정상 리셋 코드 입력 예외 검증 (Negative Test)>

Software Reset을 트리거하는 0x0001이 아닌, 정의되지 않은 예외 값(예: 0x0000, 0x0002, 0xFFFF 등)을 Reset [0xD000] 레지스터에 강제 Write (FC: 0x06) 시도한다.
예외 송신(TX) 프레임 (0xFFFF 주입 시): 01 06 D0 00 FF FF 43 E6
5초간 대기하며 펌웨어가 해당 명령을 거부하고 정상 통신을 유지하는지(재부팅이 발생하지 않는지) 확인한다.
<Phase 4. 모터 고속 구동 중 강제 리셋 및 Safe State 자가 복구 검증 (Running Reset Test)>

모터에 Run 명령을 내려 2000 RPM의 고속 회전 상태(Motor Status [0xD011] 값이 Run 상태를 지시함)를 만든다.
모터가 회전하고 있는 도중, 물리적 전원 차단 없이 Phase 2의 Software Reset 명령(01 06 D0 00 00 01 42 28)을 기습적으로 전송한다.
펌웨어가 즉각적으로 인버터 스위칭(PWM) 출력을 하드웨어적으로 차단(Coast to Stop)하여 모터가 자연 감속하며 정지하는지 물리적으로 관찰한다.
부팅 완료 후, 통신으로 Motor Status [0xD011]를 읽었을 때 모터가 스스로 재구동되지 않고, 반드시 안전한 정지 대기(Stop/Ready) 상태로 초기화되어 있는지 확인한다.

판정기준 : [Phase 1 판정 - 초기 연결]

초기 상태에서 Motor Status [0xD011] 읽기 시 에러 없이 정확한 상태 값(예: Stop = 0x0000)이 반환되어야 한다.

[Phase 2 판정 - 정상 리셋 및 부팅 시간 (통신 스택 판정 핵심)]

[중요] 리셋 명령 수신 시, 펌웨어는 무응답 상태로 바로 죽어버리는 것(Timeout 발생)이 아니라, 반드시 마스터에게 정상 Echo(01 06 D0 00 00 01 42 28)를 50ms 이내에 송신한 후에 MCU Watchdog/Soft Reset을 수행해야 합격 처리한다.
재부팅 후 통신이 다시 붙기까지의 부팅 소요 시간은 드라이브 스펙(일반적으로 2~3초 이내)을 만족해야 하며, EEPROM에 저장된 사용자 파라미터는 리셋 후에도 훼손 없이 유지되어야 한다.
[Phase 3 판정 - 예외 처리]

0x0001 이외의 값 입력 시, 펌웨어는 절대 재부팅되어서는 안 되며, 파라미터 저장을 거부하고 Modbus Exception Code 0x03 (Illegal Data Value) 프레임(예: 01 86 03 02 61)을 반환하여 방어 로직이 동작함을 증명해야 한다.
[Phase 4 판정 - 구동 중 리셋 안전성 (Safe State)]

고속 구동 중 리셋 시 하드웨어(IGBT/FET) 소손이나 과전류(OCP) 알람 발생 없이, 즉시 PWM 출력이 차단되어야 한다.
재부팅 직후 드라이브가 이전의 Run 상태를 기억하고 스스로 급가속(Auto-Restart)하는 끔찍한 결함이 절대 없어야 하며, 반드시 Stop 상태로 부팅되어 사용자의 명시적인 새로운 Run 명령을 기다려야 합격 처리한다.

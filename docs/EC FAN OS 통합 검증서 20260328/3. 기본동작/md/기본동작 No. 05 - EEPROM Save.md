# 기본동작 No. 05 - EEPROM Save

## 시험 항목

EEPROM Save

## 시험 목표

- EEPROM Save 명령(0xD000 = 0x0004) 실행 후 전원 재투입 시 변경된 파라미터가 유지됨을 확인한다.
- Save 없이 전원 재투입 시 변경된 파라미터가 이전 저장값으로 복구됨을 확인한다.

## 시험 기준

### 사전 조건

- RS485 인터페이스를 통해 드라이브와 호스트 PC 연결 완료
- Slave ID: 0x01 기준

### 시험 절차

1. RS485 인터페이스를 통해 드라이브와 호스트 PC를 연결한다.
2. 통신 툴을 실행하고 드라이브와 정상 통신되는지 확인한다.
3. 임의의 파라미터 값을 변경한다. (예: Running Direction [0xD102])
4. 변경된 값을 FC(03)으로 읽어 변경되었음을 확인한다.

---

### EEPROM Save 없이 전원 재투입

5. EEPROM Save 명령 없이 전원을 OFF/ON한다.
6. 전원 재투입 후 변경한 파라미터를 읽어 값이 이전 저장값으로 복구되었는지 확인한다.

---

### EEPROM Save 후 전원 재투입

7. 파라미터 값을 다시 변경한다.
8. EEPROM Save 명령을 전송한다.

**EEPROM Save 명령 — FC(06) Write Single Register**
- 레지스터: Reset [0xD000]
- 값: 0x0004 (EEPROM to RAM Save)

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | 0xD0 | 0x00 | 0x00 | 0x04 | CRC Lo | CRC Hi |

→ `01 06 D0 00 00 04 43 E9`

9. 전원을 OFF/ON한다.
10. 전원 재투입 후 변경한 파라미터를 읽어 EEPROM에 저장된 값이 유지되는지 확인한다.

### 판정 기준

| 케이스 | 판정 조건 |
|--------|----------|
| Save 없이 재투입 | 변경된 파라미터가 이전 저장값으로 복구됨 |
| Save 후 재투입 | EEPROM에 저장된 값이 재투입 후에도 유지됨 |

**합격 조건**: Save/No-Save 양 케이스 모두 정상 동작

시험 항목 : [RS485 Modbus 기반 EEPROM Save 제어 무결성 및 구동 중 메모리 기록(Flash Write Block) 안전성 통합 검증]

시험 목적 : Modbus FC 0x06 명령을 통한 휘발성(RAM) 파라미터 변경 및 EEPROM Save [0xD000 = 0x0004] 명령의 비휘발성 저장 동작을 확인한다. 또한, 비정상 Save 코드 입력 예외 처리, 고속 구동 중 Save 명령 인가 시 모터 제어 인터럽트(PWM) 간섭 여부 검증, 기록 중단(Power-loss) 시 메모리 오염 방어 및 자가 복구 능력을 종합적으로 확인한다.

시험 방법 : <Phase 1. 초기 상태 연결 및 휘발성(RAM) 파라미터 소멸 검증 (Base Setup & No-Save Test)>

RS485 인터페이스를 통해 드라이브(Node ID: 0x01)와 PC를 연결하고 19200bps로 통신을 개방한다.
타겟 파라미터(예: Set Point [0xD001])의 현재 값(예: 0)을 Read (FC: 0x03)하여 기록해 둔다.
Set Point [0xD001]에 0x1000 (4096) 값을 Write (FC: 0x06) 하여 메모리(RAM)에 반영시킨다.
EEPROM Save 명령을 전송하지 않고, 드라이브의 주 전원을 OFF/ON (Cold Boot) 한다.
재부팅 완료 후 Set Point [0xD001] 값을 다시 읽어(FC: 0x03), 방금 변경했던 0x1000이 완전히 소멸하고 이전 저장값(예: 0)으로 정상 복구되었는지 확인한다.
<Phase 2. 정상 EEPROM Save 동작 및 메모리 유지 검증 (Persistence Test)>

다시 Set Point [0xD001]에 0x1000 값을 Write (FC: 0x06) 한다.
이번에는 EEPROM Save 코드(0x0004)를 Reset/Control 레지스터 [0xD000]에 Write Single Register (FC: 0x06) 명령으로 전송한다.
송신(TX) 프레임: 01 06 D0 00 00 04 43 E9
드라이브의 주 전원을 완전히 차단 후 10초 대기, 다시 OFF/ON 한다.
재부팅 후 Set Point [0xD001] 값을 읽어, 0x1000 값이 비휘발성 메모리(EEPROM/Flash)에서 정상적으로 로드(유지)되는지 100% 확인한다.

<Phase 3. 비정상 Save 코드 예외 처리 및 구동 중 저장(On-the-fly Save) 스트레스 검증 (Negative & Safety Test)>

EEPROM Save로 규정된 0x0004가 아닌 정의되지 않은 쓰레기 값(예: 0x0005, 0xFFFF 등)을 [0xD000]에 강제 Write 시도하여 펌웨어의 거부 반응을 확인한다.
예외 송신(TX) 프레임: 01 06 D0 00 FF FF [맞는 CRC Hi/Lo]
[핵심 스트레스 주입] 모터에 Run 명령을 내려 2500 RPM의 고속으로 회전시킨다.
모터가 최고 부하로 구동 중인 상태에서, 임의의 파라미터를 변경하고 Phase 2의 정상 EEPROM Save 명령(01 06 D0 00 00 04 43 E9)을 기습 전송한다.
메모리를 지우고(Erase) 다시 쓰는(Write) 수십~수백 ms의 딜레이 동안, 펌웨어가 모터 제어(PWM)를 놓쳐서 모터에서 "덜컥" 거리는 이음/진동(Stuttering)이 발생하거나, MCU가 뻗어버리는지(Trip) 오실로스코프(상전류 파형)와 육안/청각으로 정밀 모니터링한다.
<Phase 4. 기록 중 강제 전원 차단(Power-Loss) 시 메모리 자가 복구 검증 (Corruption Recovery Test)>

모터를 정지시킨 후, 다시 새로운 파라미터 값(예: 0x2000)을 Write 한다.
EEPROM Save 명령(01 06 D0 00 00 04 43 E9)을 전송하는 **그 찰나의 순간(수십 ms 이내, 메모리에 기록 중인 타이밍)**에 주 전원을 강제로 뽑아버린다(Blackout).
10초 후 전원을 재투입하고 통신을 연결한다.
메모리 영역이 깨져서(Corruption) 드라이브가 하드폴트(HardFault/벽돌) 상태에 빠지지 않았는지, 그리고 CRC 무결성 체크가 실패할 경우 펌웨어가 파라미터를 '안전한 공장 초기화 값(Default)'으로 자동 복구(Fallback)하여 부팅하는지 확인한다.
판정기준 : [Phase 1 판정 - No-Save 휘발성]

Save 없이 재부팅 시, 100% 확률로 이전 값(Old Value)으로 복구되어야 하며, 찌꺼기 데이터가 남지 않아야 한다.
[Phase 2 판정 - 정상 Save 비휘발성]

0x0004 명령 전송 시 50ms 이내에 정상 Echo(01 06 D0 00 00 04 43 E9)가 수신되어야 하며, 전원 재투입 후 저장된 파라미터가 정확히 로드되어야 합격 처리한다.
[Phase 3 판정 - 예외 처리 및 구동 중 저장 안정성 (소프트웨어 아키텍처 핵심)]

0x0005 등 비정상 코드 입력 시, 즉시 **Modbus Exception Code 0x03 (Illegal Data Value)**로 응답해야 한다.
[매우 중요] 고속 구동 중 Save 명령을 날렸을 때, Flash Write Delay로 인해 모터 제어 인터럽트가 블록(Block)되어 **모터가 출렁이거나 하드웨어 과전류(OCP) 알람이 발생하면 즉각 불합격(Fail)**이다. 메모리 저장을 백그라운드나 안전한 슬롯에서 처리하여 모터 구동에 절대 영향을 주지 않아야 합격이다.
[Phase 4 판정 - 메모리 오염 자가 복구]

저장 중 전원 상실 스트레스에도 불구하고, 재부팅 시 절대 펌웨어 데드락(벽돌) 현상이 없어야 한다.
[0xD001]을 읽었을 때 0xABCD 등 치명적인 오동작을 유발할 수 있는 쓰레기 값이 올라오면 불합격이며, 펌웨어가 자체적으로 오염을 감지하고 이전 값 또는 **Factory Default 값으로 덮어씌워 시스템을 안전하게 방어(Fallback)**해야만 최종 합격 처리한다.

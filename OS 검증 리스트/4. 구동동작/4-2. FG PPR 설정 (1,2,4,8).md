# 4-2. FG PPR 설정 (1, 2, 4, 8)

| 항목 | 내용 |
|------|------|
| **분류** | 구동동작 |
| **No** | 4-2 |

## 시험 목적

FG(Frequency Generator) PPR(Pulse Per Revolution) 설정에 따라 속도 피드백 신호가 정상적으로 동작하는지 확인한다.

## 적용 모델

- EC-FAN ✓

## 시험 장비

- EC FAN 1EA
  - (Node Address를 1로 설정한 상태에서 시험한다. 설정 방법은 RS485 No. 1-3 시험 참조)
- USB to RS485 Converter
- Hercules v3.2.8 (테스트 프로그램)
- 오실로스코프 (FG 출력 파형 확인용, 선택)

## 시험 방법

1. USB to RS485 Converter를 이용하여 EC FAN과 노트북을 연결한다.
2. Hercules v3.2.8 테스트 프로그램을 실행 후 통신 설정을 진행한다.
   - Baud: 19200bps, Data Size: 8, Parity: even
3. Open 버튼을 클릭하여 COM Port 연결을 확인한다.
4. Open-loop 모드로 팬을 일정 속도로 구동한다.
5. PPR을 1로 설정한다.

**PPR 파라미터 쓰기**

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | PPR Addr Hi | PPR Addr Lo | 0x00 | PPR Value | CRC Hi | CRC Lo |

- PPR 값: 1 → 0x0001, 2 → 0x0002, 4 → 0x0004, 8 → 0x0008

6. FG 출력 단자에서 펄스 주파수를 오실로스코프로 측정하여 예상 주파수와 일치하는지 확인한다.
7. PPR을 2, 4, 8로 순서대로 변경하며 각각 확인한다.
8. Actual Speed [0xD02D] 파라미터가 PPR 변경과 무관하게 정확한 속도를 출력하는지 확인한다.

## 판정기준

- 각 PPR 설정값에 따라 FG 출력 펄스 주파수가 정확히 해당 배율로 출력됨
- Actual Speed 파라미터가 PPR 변경과 무관하게 실제 속도를 정확히 반영

## 시험결과

| PPR 설정 | FG 주파수 (기대) | FG 주파수 (실측) | 판정 |
|----------|----------------|----------------|------|
| 1 | - | - | 합격 / 불합격 |
| 2 | - | - | 합격 / 불합격 |
| 4 | - | - | 합격 / 불합격 |
| 8 | - | - | 합격 / 불합격 |

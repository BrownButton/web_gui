# 1-1. 기본 연결 동작 시험 및 Baudrate, Parity 변경

| 항목 | 내용 |
|------|------|
| **분류** | RS485 |
| **No** | 1-1 |

## 시험 목적

EC FAN과 RS485 통신 연결이 정상적으로 이루어지는지 확인하고, Baudrate 및 Parity 변경 기능을 검증한다.

## 적용 모델

- EC-FAN ✓

## 시험 장비

- EC FAN 1EA
- USB to RS485 Converter
- EC FAN Configurator (EC FAN Control 프로그램)

## 시험 방법

### 기본 연결 동작 확인

1. USB to RS485 컨버터를 이용하여 EC FAN과 노트북을 서로 연결한다.
2. EC FAN Control 프로그램을 실행한다.
3. COM Port: USB to RS485 Converter 접속 포트, Baudrate: 19200, Parity: Even으로 설정 후 접속 버튼을 누른다.
   - (Baudrate 19,200bps, Even Parity는 공장 출하 상태의 초기값)
4. Device, SW Ver, Boot Ver이 정상 출력되는지 확인한다.

### Baudrate 변경 시험

5. Setting 메뉴에 진입하여 Baudrate를 다른 값으로 변경한다. (예: 38400bps)
6. Save to Memory 버튼을 눌러 파라미터를 저장한다.
7. 전원 재투입 후 변경된 Baudrate로 재접속한다.
8. Device, SW Ver, Boot Ver이 정상 출력되는지 확인한다.
9. 다시 기본값(19200bps)으로 변경 후 저장한다.

### Parity 변경 시험

10. Setting 메뉴에 진입하여 Parity를 변경한다. (예: None 또는 Odd)
11. Save to Memory 버튼을 눌러 파라미터를 저장한다.
12. 전원 재투입 후 변경된 Parity로 재접속한다.
13. Device, SW Ver, Boot Ver이 정상 출력되는지 확인한다.
14. 다시 기본값(Even)으로 변경 후 저장한다.

## 판정기준

- 기본 접속 시 Device, SW Ver, Boot Ver이 정상 출력
- 변경된 Baudrate로 재접속 시 정상 통신 연결
- 변경된 Parity로 재접속 시 정상 통신 연결

## 시험결과

- [ ] 합격 / 불합격

## 기타

- Baudrate 변경 후 반드시 기본값(19200bps)으로 복구할 것
- Parity 변경 후 반드시 기본값(Even)으로 복구할 것
- 변경된 통신 설정과 드라이브 설정이 일치하지 않으면 접속 불가

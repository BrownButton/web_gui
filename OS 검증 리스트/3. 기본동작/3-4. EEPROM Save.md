# 3-4. EEPROM Save

| 항목 | 내용 |
|------|------|
| **분류** | 기본동작 |
| **No** | 3-4 |

## 시험 목적

파라미터 변경 후 EEPROM Save 명령을 실행하면 전원 재투입 후에도 변경된 파라미터가 유지되는지 확인한다.

## 적용 모델

- EC-FAN ✓

## 시험 장비

- EC FAN 1EA
  - (Node Address를 1로 설정한 상태에서 시험한다. 설정 방법은 RS485 No. 1-3 시험 참조)
- USB to RS485 Converter
- Hercules v3.2.8 (테스트 프로그램)

## 시험 방법

1. USB to RS485 Converter를 이용하여 EC FAN과 노트북을 연결한다.
2. Hercules v3.2.8 테스트 프로그램을 실행 후 통신 설정을 진행한다.
   - Baud: 19200bps, Data Size: 8, Parity: even
3. Open 버튼을 클릭하여 COM Port 연결을 확인한다.
4. 임의의 파라미터 값을 변경한다. (예: Set Point [0xD001], Running Direction [0xD102] 등)
5. 변경된 값을 FC(03)으로 읽어 변경되었음을 확인한다.

### EEPROM Save 없이 전원 재투입

6. EEPROM Save 명령 없이 전원을 OFF/ON한다.
7. 전원 재투입 후 변경한 파라미터를 읽어 값이 초기화되었는지 확인한다.
   - (EEPROM Save 없이 전원 재투입 시 이전 저장값으로 복구되어야 함)

### EEPROM Save 후 전원 재투입

8. 파라미터 값을 다시 변경한다.
9. EEPROM Save 명령을 전송한다.

**EEPROM Save 명령 (0xD000 레지스터에 Save 코드 쓰기)**

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x06 | 0xD0 | 0x00 | EEPROM Save Code Hi | EEPROM Save Code Lo | CRC Hi | CRC Lo |

10. 전원을 OFF/ON한다.
11. 전원 재투입 후 변경한 파라미터를 읽어 EEPROM에 저장된 값이 유지되는지 확인한다.

## 판정기준

- EEPROM Save 명령 없이 전원 재투입 시: 이전 저장값으로 복구
- EEPROM Save 후 전원 재투입 시: 변경된 값이 유지됨

## 시험결과

| 시험 | 판정 |
|------|------|
| Save 없이 전원 재투입 시 값 초기화 확인 | 합격 / 불합격 |
| Save 후 전원 재투입 시 값 유지 확인 | 합격 / 불합격 |

## 기타

EEPROM Save 전/후 파라미터 값 및 전원 재투입 후 데이터 캡처 첨부

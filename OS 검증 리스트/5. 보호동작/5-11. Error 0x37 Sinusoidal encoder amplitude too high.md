# 5-11. Error 0x37 Sinusoidal Encoder Amplitude is Too High

| 항목 | 내용 |
|------|------|
| **분류** | 보호동작 |
| **No** | 5-11 |

## 시험 목적

사인파 엔코더(Sinusoidal Encoder) 신호 진폭이 최대 임계값을 초과했을 때 보호 동작(알람)이 수행되는지 확인한다.

## 적용 모델

- EC-FAN ✓

## 시험 장비

- EC FAN 1EA
  - (Node Address를 1로 설정한 상태에서 시험한다. 설정 방법은 RS485 No. 1-3 시험 참조)
- USB to RS485 Converter
- Hercules v3.2.8 (테스트 프로그램)

## 시험 방법

> ⚠️ **참고**: 검증 실험 가능 여부 확인 후, 불가 시 삭제 처리

1. USB to RS485 Converter를 이용하여 EC FAN과 노트북을 연결한다.
2. Hercules v3.2.8 테스트 프로그램을 실행 후 통신 설정을 진행한다.
   - Baud: 19200bps, Data Size: 8, Parity: even
3. 엔코더 신호 진폭을 과도하게 높이는 조건을 구현한다.
   - (엔코더 자석을 교체하거나 홀센서 거리를 지나치게 가깝게 설정)
4. 전원을 인가한다.
5. Motor Status [0xD011]을 읽어 에러 비트 Set 여부를 확인한다.

**Motor Status 읽기 명령**

| 바이트 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|--------|------|------|------|------|------|------|------|------|
| 내용 | 0x01 | 0x04 | 0xD0 | 0x11 | 0x00 | 0x01 | 0x59 | 0x0F |

6. Error 코드 파라미터를 읽어 0x37 (Sinusoidal Encoder Amplitude Too High)가 발생했는지 확인한다.

## 판정기준

- 엔코더 신호 진폭 과대 시 보호 동작을 수행함
- Error 코드 0x37 확인

## 시험결과

- [ ] 합격 / 불합격 / 시험 불가

## 기타

- **검증 실험 가능 여부 확인 후 불가 시 항목 삭제 처리**
- 시험 후 드라이브를 원상 복구하고 Auto Calibration을 다시 실시한다.
